## 文档边界

本文件负责回答“下一步做什么、为什么做、怎样算完成”。已完成事实进入 `../CHANGELOG.md`，长期技术规则进入 `01_开发者手册.md`，全局设计准绳进入 `03_项目全局设计规划.md`；世界观和纵切设计的细节以 `07_江南雨巷纵切规格.md`、`10_世界观构建方案.md` 和 `11_项目设计记忆快照.md` 为支撑依据。`13_江南区域新版大地图设计方案.md` 是在 `07` 基线上的章节级扩展优化草案，不直接覆盖 `07`，但作为后续改版 `07`、扩展内容 Schema 和搭建编辑器/内容库的方向依据。

## 背景与问题

项目已完成 M0 浏览器原型、内容数据样例、Node 静态测试、Python 内容校验和前期设计文档，并已把 `07_江南雨巷纵切规格.md` 扩展为江南第一版投放设计书。`07` 的价值在于验证纸伞门径、雨巷主题、NPC/Boss/四维选择、工艺链和内容对象拆分；它适合 45-60 分钟纵切与 M1 白盒，但不足以支撑正式端游章节的体量。

`13` 的判断是：江南不应只是线性节点纵切，而应在 `07` 基础上扩容为不少于 5 小时、6 个大型子区域、多点位、可回访、可自给循环的正式章节。下一阶段的核心不是抛弃 `07`，而是完成“基线继承、结构扩容、玩法加密、生产前置”：把 `07` 的 11 个节点映射进 `13` 的六大区，并在继续开发可玩内容前，先建立标准化内容库、人物/地块/交互/工艺等编辑器蓝图和校验规则。

## 阶段目标

1. M0 demo 能稳定运行并通过自动测试、内容校验和人工浏览器冒烟。
2. 保留 `07` 作为 M1 纵切和内容数据基线，完成 `07` 11 个节点到 `13` 六大区内部点位的迁移说明。
3. 明确江南章节化扩展目标：首次通关不少于 5 小时，内容拆为主线、百工探索、战斗挑战、大地图探索、日常回访五类。
4. 建立内容库优先策略：新增人物、地块、建筑、采集、工艺、敌人、Boss、支线和资源引用先进入 `content/` 与 Schema，再接入 demo 或未来 Runtime。
5. 定义标准化编辑器的最低产出格式、对象边界和校验规则，避免继续用一次性脚本堆内容。
6. 后续大地图白盒最小验证应至少覆盖 2 个大区、若干点位，以及采集、工艺、订单、战斗、回访的闭环。

## 依赖与风险

- 设计口径风险：`13` 仍是新版区域草案，不能在没有迁移图和验收口径时直接覆盖 `07` 的当前 M1 基线。
- 范围膨胀风险：从 45-60 分钟扩到 5 小时，会显著增加策划、文本、美术、音频、程序和 QA 负担。
- Schema 迁移风险：线性节点模型扩成开放大区后，现有 RegionDef 索引、测试和校验器需要同步升级。
- 编辑器债务风险：如果先做漂亮工具而不定义文本化输出、稳定 ID 和校验规则，编辑器会变成一次性白盒。
- 资源规模风险：6 大区、多个小 Boss、长期循环和回访状态需要更严格的资产 manifest、预览、压缩和复用策略。
- 技术表达风险：Three.js、Canvas 或未来 Axmol 只能承担表现和预览，不能拥有任务推进、玩法判定和内容引用。
- 文化资料风险：纸伞、染纸、灯市、行会、水镇空间、舟修和桥工等内容仍需按 `08_资料索引与调研计划.md` 二次核验。

## 不做事项

- 本阶段不做完整全国大地图。
- 本阶段不做 MMO 或实时联机。
- 本阶段不引入大型二进制编辑器作为内容唯一源头。
- 本阶段不把当前 browser demo 误认作最终 Axmol 实现。
- 本阶段不继续把新增人物、地块、采集点、支线或 Boss 只写进场景脚本。
- 本阶段不在没有迁移图的情况下删除或覆盖 `07` 纵切基线。
- 本阶段不直接复制大型第三方依赖进仓库；正式接入前先写 ADR。

## 验收方式

- 自动测试：运行 `npm test`。
- 内容校验：运行 `npm run validate:content`。
- 浏览器冒烟：运行 `npm run dev`，打开 `http://127.0.0.1:4174/`。
- 人工路径：检查加载非空、移动、冲刺、架伞、收风、借风、交互、夜市选择、工艺白盒、归位和重置。
- 规划检查：能说明 `07` 的 11 个节点如何进入 `13` 的六大区，且 5 小时章节目标被拆成主线、百工探索、战斗挑战、大地图探索和日常回访。
- 数据检查：大地图 Schema 草案能表达大区、点位、连接、门槛、刷新、资源循环、小 Boss、回访状态和资源引用。
- 文档检查：README 能导航到规划、开发者手册、测试指南、07/13 设计和发布历史；已完成事实写入 `CHANGELOG.md`。

## 待确认问题

- 是否正式进入“江南章节化扩展准备期”，并将 `13` 作为后续改版 `07` 的主依据。
- 大地图内容 Schema 是否采用 JSON Schema 文件、TypeScript 类型生成，还是先扩展 Python 校验器。
- 第一版标准化编辑器先做内部 Web 表单、JSON 模板生成器，还是结合浏览器白盒面板逐步演进。
- `07` 11 个节点迁移到 `13` 六大区时，哪些保留为主线必经点，哪些降级为教学、支线或回访点。
- 大地图白盒最小验证优先选择哪两个大区：建议从烟雨竹巷与河埠水市开始，因为它们能覆盖作坊、NPC、市场、订单和水路。
- Axmol WebAssembly、Three.js 高精度预览和正式资源 manifest 的首个技术冒烟要覆盖哪些输入、音频、碰撞和性能指标。
- 是否需要在正式接入第三方依赖前建立许可扫描和依赖版本记录。
