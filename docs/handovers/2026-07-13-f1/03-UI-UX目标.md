# UI/UX 目标｜F1 可访问 DOM 流程原型

> 任务：`F1-UI-01A`；Owner：UX 负责人；必审：UI、美术指导、Web、Gameplay、可访问性、QA。
> 分支：从本交接包提交建立独立 worktree `codex/f1-ui-ux`；代码快照基线为 `9991a3d`。
> 写入边界：新建 `prototypes/f1-ui-ux/`、08 与本任务证据；第一阶段不修改 C++、Profile、IndexedDB 或现有主开发宿主文件。
> 最后更新：2026-07-13。

## 目标

建立可点击、可键盘、可读屏、可响应式验证的 F1 DOM UI 原型，覆盖标题到结算/刷新恢复的完整信息路径，并冻结 UI 状态/事件接口。原型只消费模拟的只读 `UiEvent`/面板模型并发出 Action，不计算战斗、任务、奖励或保存结果。

## 权威输入

- 02 §3/3.1/4.3 的 `F1-UI-01`，08 §1–17，10 §8/21，16 §6/15。
- `content/design/action-registry.json` 是输入命名真相；新增 Action 先提合同变更，不在 UI 私造键位。
- `prototypes/f1-visual-sample` 只提供镜头、色彩与 HUD 占位方向；其 Phaser 状态不是正式 UI 数据源。
- 美术基线未锁定前使用可替换设计令牌和占位图，不抢先把临时素材写死。

## 第一阶段交付

1. 在隔离目录实现标题/能力检查、真实加载阶段、Guest 新建/继续/导入入口、设置/辅助入口。
2. 实现战斗 HUD：生命、体力/定力、工灯、架势、Boss/公共工程、最多三条短目标、证据短堆栈和情境交互；主战场始终占绝对主体。
3. 实现对话、短教程、证据发现、暂停、失败/重试、重大选择、双结算和返回标题流程。
4. 明确保存文案状态：等待、保存中、已保存在此设备、离线但本地已存、尚未安全保存/可重试；禁止未确认事务显示“已保存”。
5. 用固定 fixtures 驱动正常、低血、Boss 换相、失焦、存储慢/失败、重复奖励被去重、证据不足/完整归位两结算等状态。
6. 建立设计令牌、组件状态表、焦点图和 DOM/Canvas 投影接口说明；不把 CSS/DOM 布局对象传进 GameCore。

## 交互与可访问验收

- 纯键盘走完核心流程；Tab 顺序、焦点回退、Escape、IME、页面失焦/恢复没有输入泄漏。
- 1080p、1440p、4K、16:10、21:9、窄屏提示和 200% UI 无阻断溢出；地址栏/DPR/缩放变化不重建 Session。
- 屏幕阅读菜单路径、静音+重要声音字幕、高对比、减震、色觉和延长窗口状态都有可验证入口。
- DOM `pointer-events` 只在真实控件启用；HUD 不吞战斗输入，不用 Canvas 栅格化长文本规避语义化 DOM。
- 浏览器自动化和截图覆盖首屏、战斗、失败、选择、结算、保存失败和 200% UI；记录控制台/页面错误。

## 集成交付

第一阶段只交回隔离原型、fixtures、接口表和证据。主开发奖励事务提交后，再由技术负责人另开集成任务把批准组件接到 `apps/web-shell/shell.html`；不得在本角色分支直接接管 C++ 或 IndexedDB 事务。

## 非目标

不做网页大厅、商城、完整背包/地图/天工录、三地区 UI 或最终品牌包装；不把一次性样例 HUD 当作已完成正式组件。
