# 《天工渡 / Heavenwrights》前期设计总览

> 版本：0.5  
> 日期：2026-06-23  
> 状态：当前有效前期方向  
> 关联：`11_项目设计记忆快照.md`

## 1. 当前定位

《天工渡》是一款 Web 端优先的 2D / 类 2.5D 像素风动作 RPG。玩家作为“天工渡人”，在斜俯视动作区域、城镇据点、工坊、商路节点和 Boss 区之间行走，通过拜师学艺、理解工艺、采集材料、打造器具、掌握门径，在战斗、探索、解谜和地区选择中让器物、场所和人重新归位。

《造梦西游》《造梦无双》只作为“网页媒介、关卡推进、角色成长、Boss 挑战”的体感参考；《暖雪》可作为“像素风、斜俯视动作平面、层次化场景和高反馈战斗”的视觉与空间参考。《天工渡》的核心差异是：

- 非遗工艺转化为动作与装备机制。
- NPC 师承承担成长与情感连接。
- 地区四维影响剧情、经济、敌人和结局反馈。
- 战斗终点是修复、净化、唤醒、归位，而不是杀戮。

## 2. 北极星

> 让“工艺”成为动作，让“师承”成为成长，让“归位”成为战斗终点。

这句话用于判断内容是否跑偏：

- 非遗内容如果只停留在图鉴说明，还没有进入核心玩法。
- 成长如果只是数值变大，还没有形成门径、器具、师承或动作变化。
- 战斗如果只是在消灭怪物，还没有完成“失序 -> 归位”的项目表达。

| 项目 | 当前方案 |

| 平台 | Web 端优先，桌面端作为开发调试与后续封装方向 |

| 首个纵切 | 江南雨巷 + 纸伞门径 |

## 4. 设计支柱

### 4.1 2D / 类 2.5D 动作底盘

统一基础输入：

- 平面移动。
- 闪避 / 冲刺。
- 轻击。
- 重击。
- 格挡 / 弹反。
- 技艺键。
- 门径技能键。
- 交互 / 采集 / 对话。

跳跃、翻越、借风越行不作为所有场景的通用核心输入，而作为地形交互或门径能力出现。不同门径改变攻击距离、位移方式、资源循环、防御节奏、破绽处理、地形和机关解法。

### 4.2 工艺进入战斗、探索和叙事

纸伞不是“做伞小游戏”，而是：

- 伞骨影响格挡稳定。
- 伞面影响借风越行和风息积累。
- 糊纸影响弹反窗口或耐久缺陷。
- 师傅评价影响技能节点。
- 商业急单影响市场、传承和 Boss 结局。

### 4.3 地区包驱动长期内容

每个地区包至少包含：

- 一个城镇 / 据点。
- 一组 2D / 类 2.5D 区域节点。
- 一条主线冲突。
- 一到两种代表技艺。
- 一组 NPC 与师承关系。
- 一条材料 / 工艺链。
- 普通敌人、精英敌人、Boss。
- 可回访支路。
- 四维选择与地区回声。

### 4.4 数据驱动与 AI 协作

内容以文本化数据为公开接口：

- `RegionDef`
- `StageDef`
- `RoomDef`
- `NpcDef`
- `DialogueDef`
- `PathwayDef`
- `SkillDef`
- `EnemyDef`
- `BossDef`
- `CraftProcessDef`
- `MaterialDef`
- `ArtifactDef`
- `WorldEventDef`

AI 可以参与生成、审查和修订；工具链负责校验引用、数值范围、资源缺失和可玩流程。

### 4.5 执灯渡人与一卷未完成的《天工录》

玩家的长期驱动力不是被动救火，而是以年轻执灯渡人的身份，持临时渡牒与工灯，带着一卷未完成的《天工录》行脚九州。

每个地区都是《天工录》的一条活页：

- 记录地方技艺和新工法。
- 见证师承、工评、节庆和商路。
- 用工灯检校地方工印、工坊器物和工事异象。
- 在护送、巡检、试艺、采集和工事关卡中持续战斗。
- 处理工坊、行会、祠社和器物牵出的祟局。
- 补全师傅旧页留下的线索。
- 逐渐理解不同地区反复出现的“无款工印”。

这条主线既给玩家个人目标，也给长期地区更新提供自然入口。

## 5. 英文名

暂定英文名 `Heavenwrights` 保留。

`wright` 有“匠人、制造者”的意味，适合“天工”与“修复天序”的概念。正式使用前仍需做商标、域名、Steam、社媒账号和发行区域检索。
