# 13｜编辑器与模板生产

> 上级文档：[`../01-开发者文档.md`](../01-开发者文档.md)
> 状态：Target Specification；F1 实现最小闭环
> 最后更新：2026-07-10

## 1. 原则

编辑器不是“方便时再做”的附属品，而是大型长期内容的生产系统。每个正式模板必须支持创建、校验、预览、差异、复制、重命名显示名、迁移和批量检查；不要求作者打开代码才能完成标准内容。

编辑器与命令行使用同一 Contracts、Schema、默认值、校验器和烘焙器。UI 只提供可视化和引导，不能形成第二套真相。

## 2. 形态

主编辑器作为 Godot EditorPlugin 集合，利用场景、资源、相机和运行预览。大型表格、批量文本和报告可由外部 Web/桌面工具补充，但保存格式不变。

```text
Tgd.Editor.Godot
├── ContentBrowser
├── InspectorAdapters
├── CharacterStudio
├── CombatLab
├── BossGraph
├── QuestGraph
├── DialogueGraph
├── RegionWorkbench
├── CraftWorkbench
├── ValidationCenter
└── BuildAndPackagePanel
```

## 3. 内容浏览器

按类型、包、地区、阶段、负责人、标签、错误和最近修改筛选。每项显示稳定 ID、显示名、阶段、拥有包、依赖数、被引用数和校验状态。

关键操作：从模板新建、批量验证、显示引用/反向引用、比较版本、移动文件但保持 ID、弃用、生成迁移、打开预览、定位场景实例。删除前显示存档/包/任务影响，默认改为弃用而非立即删除。

## 4. 角色工作室

### 4.1 新建向导

1. 选择 NPC/敌人/Boss/可交互器物原型。
2. 选择拥有包、地区、内容阶段和负责人。
3. 工具生成稳定 ID、文件、最小组件组合、本地化 key 和测试夹具。
4. 选择表现占位（体型、碰撞、动画档、语音档），不复制最终资产。
5. 立即执行 Schema/引用/预算校验并进入沙盒。

### 4.2 核心 NPC 页签

- **Identity**：一句话、角色作用、职业、组织、显示与表现引用。
- **Schedule**：日历/天气/章节时间轴，地点锚点和异常覆盖可视化。
- **Relationship**：三阶段关系弧、维度条件、关键共同经历和拒绝/恢复。
- **Interactions**：指导、交易、工艺、同行、研究、谈判等通用能力。
- **Narrative**：对话、任务、传闻、证据和天工录引用。
- **World Response**：地区归位、选择和跨区变量的状态矩阵。
- **Diagnostics**：不可达状态、冲突覆盖、缺失本地化、预算和引用。

### 4.3 沙盒预览

一键启动小型场景，可选择时间、天气、关系阶段、任务状态和地区状态。预览日程切换、交互、对话条件、战斗/同行、保存/读档和缺席代理。所有临时覆盖清楚标记，不写回内容。

一个灰盒核心 NPC 从模板到可预览目标中位时间：熟练作者 30 分钟内；完整角色仍需真实设计、文字、资产和评审，工具速度不等于内容完成。

## 5. 战斗实验室

可加载任意角色、武器、能力、敌群和场地档，控制固定种子、帧率、输入脚本和辅助选项。时间轴显示命令、成本、移动、命中窗、效果、取消窗、事件和表现偏差。

支持：逐 Tick、慢放、碰撞/导航可视化、伤害/定力/失序图、AI 意图、威胁令牌、工灯证据、性能采样、回放差异。作者可以调整内容参数并热重载灰盒，但结构/Schema 变化需重新烘焙。

## 6. Boss 阶段图

节点代表阶段，边代表条件与过渡。侧栏编辑招式集、AI 权重、环境状态、证据、音乐/演出标签、重试点和归位结果。

静态检查：

- 起点唯一、至少一个完整归位终点、所有节点可达。
- 不存在无出口的非终止阶段和无上限循环。
- 生命阈值、证据、环境与时间条件没有永远矛盾。
- 每阶段至少有一个可读招式变化或场景决策变化。
- 重试快照覆盖所有权威机关与同伴。
- 辅助模式可到达所有必需终点。

一键生成阶段遍历测试和最小回放脚本。

## 7. 任务与对话图

任务图只提供受控节点类型；拖线时显示类型兼容、事件语义和持久字段。编辑器可以模拟事件序列、跳到任意节点、随机遍历和显示不可达/互斥分支。

对话图与任务图通过领域事件连接，不直接互相修改内部状态。对话预览支持关系、称谓、本地化长度、语音时长、可打断、移动对话和字幕。批量文字编辑可导出稳定行 ID 的表格，再安全合并。

## 8. 地区工作台

叠加显示 Cell、流式门、导航区、相机约束、地标、任务路径、NPC 日程锚点、遭遇槽、音频区、天气、归位状态和性能热图。

预算实时汇总：首轮时长、路径长度、战斗/社交/工艺比例、兴趣点类型、实体/AI/灯光/粒子、内存和包依赖。超预算是错误或显式豁免，不能只在最后优化。

工作台能切换“新到、章节中、已归位、不同重大选择”状态，差异资源必须复用基础场景而非复制整个地图。

## 9. 工艺与武器工作台

以工艺域和用途权衡组织，不只编辑最终数值。部件图显示兼容接口、材料、工序、属性曲线、视觉插槽、损坏/维修和解锁来源。能力时间轴与战斗实验室共享。

工作台可自动生成对比场景：基础部件、极端硬度/韧性、非法组合、低资源、不同天气和目标材质，防止只调一个理想用例。

## 10. 模板注册表

当前权威注册表位于 [`../../content/templates/template-registry.json`](../../content/templates/template-registry.json)，结构由 [`../../content/schemas/template-registry.schema.json`](../../content/schemas/template-registry.schema.json) 约束。

每个模板声明：

```text
templateId / templateVersion
contentType / compatibleSchemaRange
requiredModules / optionalModules
defaults / generatedFiles / namingRules
previewScene / validators / testGenerators
migrationFrom / owner / documentationLink
```

模板升级不自动静默改所有实例。工具生成预览差异，作者选择采用新默认或保留显式值；结构变化通过批量迁移。

F1 最小模板：核心 NPC、普通敌人、章节 Boss、任务链、对话、武器能力、子地区。M1 前增加地区 Boss、商店、工艺配方、公共工程和世界事件。

## 11. Undo、并发与文件策略

- 所有 UI 修改进入 Undo/Redo 事务；跨文件操作要么全部成功，要么回滚。
- 保存使用临时文件 + 原子替换，并在磁盘变更时提示三方比较。
- 内容按稳定对象拆分，避免多人频繁改同一个巨型 JSON。
- 图节点使用稳定 node ID，重排不造成语义 diff。
- 规范化序列化保证键序、换行和数值格式一致。
- 二进制 Godot 场景冲突优先通过场景分片和资源组合避免，不依赖人工合并。

## 12. 验证中心

统一展示 Schema、引用、语义、预算、本地化、资产、性能和测试结果。错误可点击定位字段/场景节点；支持按包、负责人和严重性筛选，并导出 CI 同格式报告。

校验器支持增量模式和全量模式。保存时只跑快速规则；提交前跑包级；CI 跑全量和沙盒；发布跑确定性双烘焙、签名与安装回归。

## 13. AI 辅助

AI 面板只能通过受控命令工作：基于模板生成草案、补齐测试变体、解释校验错误、生成本地化占位、比较重复内容。调用时传入最小相关上下文和允许 ID 列表，输出进入预览差异，不直接写主文件。

高风险操作（新正史、长期 API、批量覆写、删除、迁移、正式台词定稿）必须人工确认。每条 AI 变更记录提示版本、输入摘要、负责人和审查状态，不保存敏感对话全文到发布包。

## 14. Headless 与 CI

任何编辑器动作都要有等价命令行能力：

```text
tgd-content create --template core_npc --package region_jiangnan --name "..."
tgd-content validate --package region_jiangnan --profile ci
tgd-content bake --package region_jiangnan --deterministic
tgd-content test --fixture npc_shen_yan --scenario post_resolution
tgd-content report --budgets --references --localization
```

真实命令名在 F1 实现时锁定。GUI 只编排这些服务，确保无界面构建机可完整工作。

## 15. 编辑器自身质量门禁

- 5000 个内容对象下浏览、搜索和增量校验无明显阻塞。
- 崩溃或验证失败不损坏源文件；能恢复未保存事务。
- 新作者按教程在 90 分钟内创建并预览角色、任务和简单 Boss。
- 编辑器生成内容与命令行烘焙哈希一致。
- Undo/Redo、外部修改、合并冲突和模板迁移有自动测试。
- 插件升级能打开上一个正式版本的全部内容。

“作者可用”必须由内容团队盲测，不由工具作者自评。
