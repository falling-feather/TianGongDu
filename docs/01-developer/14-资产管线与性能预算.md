# 14｜资产管线与性能预算

> 上级文档：[`../01-开发者文档.md`](../01-开发者文档.md)
> 状态：Baseline；具体视觉组合由 F1 Render Spike 锁定
> 最后更新：2026-07-10

## 1. 资产原则

- 源资产、导入设置、运行时产物和发布包分层；任何发布资源都能从源和工具版本重建。
- 先定屏幕贡献和性能档，再定面数/分辨率；不用“越大越好”。
- 同类资产共享材质、骨骼、动画、碰撞与 LOD 规范。
- 视觉差异来自轮廓、材质、动作和工艺逻辑，不靠无限独占 Shader。
- 预算从灰盒开始持续显示，不能在 Beta 才第一次测。

## 2. 目录与命名

```text
assets_src/<domain>/<region>/<asset_id>/
  source.blend / source.kra / source.aseprite
  textures/
  references/
  license.yaml
  import.yaml

assets_runtime/<package>/<asset_id>/
  generated resources
  manifest fragment
  thumbnails / previews
```

资产 ID 使用 `region_type_meaning_variant`；文件名英文小写下划线。显示名和作者信息进入 metadata，不进入路径。导入器禁止依赖绝对路径或作者机器插件。

## 3. 2.5D 表现烘焙

F1 比较并锁定：3D 场景 + 3D/2D 混合角色、3D 场景 + 2D 骨骼角色、全 3D 风格化角色三种方案。评估维度：近远景一致、遮挡/灯光、动画吞吐、换装、碰撞、镜头、性能、内容工具和团队能力。

无论最终组合如何，Gameplay 只暴露实体快照、碰撞查询和表现标签。角色资产可替换，不改变战斗规则和存档。

## 4. 模型管线

1. 统一米制、轴向、原点、比例和命名。
2. 高模/低模或风格化直接建模，记录目标屏幕尺寸。
3. UV、材质槽、骨骼、碰撞代理、LOD 和 Socket 自动检查。
4. 导出 GLB/glTF 或 Bake 确认格式，导入设置由 `import.yaml` 生成。
5. 自动生成缩略图、转台、三角形/材质/骨骼报告。
6. Godot Headless 导入并加载验证；失败不提交运行时产物。

静态小物优先合批/实例化；大型建筑按 Cell 和遮挡拆分，不按 DCC 作者习惯随意切。可交互部件与视觉网格分离。

## 5. 角色与动画

- 体型/骨骼家族受控，核心 NPC 可有独特轮廓但优先共享基础骨架和动作层。
- 动画分 locomotion、combat、profession、social、cinematic；职业动作是角色辨识核心，不只做站立聊天。
- Root Motion 是否用于某类能力由 Runtime 明确合同控制；规则位移不能只存在动画中。
- 动画事件仅发声音/粒子/脚步等表现标签，命中和资源由 Runtime 时间轴决定。
- 每个动作有首帧、关键姿态、接触、恢复、取消窗和减速预览。
- 面部与口型按语音预算分级：主线近景、核心 NPC 常规、支持 NPC 简化。

## 6. 纹理与材质

纹理按屏幕尺寸和复用分 256/512/1024/2048/4096 档；4096 只用于大面积或关键角色图集且需审查。颜色、法线、ORM/掩码和特殊通道命名固定。

材质基于有限主 Shader：环境、角色、纸/布、金属、半透明、水面、特效。地区变体通过参数和纹理，不复制整套 Shader。Shader 变体数量和编译缓存纳入 CI 报告。

## 7. 2D、UI 与字体

- UI 使用矢量/九宫格/多分辨率图集组合，避免每分辨率一套位图。
- Sprite 图集按包、过滤模式、压缩和加载生命周期分组，不能跨所有地区做单一巨图。
- 字体必须确认简中字符覆盖、许可、Hinting 和回退；发布包只裁剪实际字符并保留动态文本策略。
- 概念图和参考图不直接进入发布资源，除非完成授权和格式处理。

## 8. VFX

特效必须服务读招、命中、工灯证据、环境状态或气氛。读招颜色/形状在地区间保持语义，装饰层可以变化。粒子提供低/中/高档，关键提示不因低档完全消失。

屏幕闪光、抖动、拖影和高亮响应 08 的辅助设置。透明过绘、实时灯、碰撞粒子和 GPU 粒子数量在场景报告中可见。

## 9. 音频

音频总线：Master、Voice、Music、Ambient、SFX、Combat、UI、Accessibility。事件使用稳定音频 ID 和参数，不从 Gameplay 拼文件路径。

环境采用分层声景和区域/天气/时间状态，不为每个 Cell 烘焙长立体声。战斗声音要区分威胁方向、材质、精确格挡、破势、证据与归位。关键声纹玩法有视觉/触觉等价。

语音文件带角色、行 ID、语言、版本、时长、响度和许可；台词锁定后再批量录制。流式长音频与驻留短 SFX 分开打包。

## 10. 帧与内存预算

目标档：1080p、推荐硬件、60 FPS。F1 使用代表性而非空场景验证。

| 项目 | Release P95 目标 | 硬门 |
| --- | ---: | ---: |
| 总 CPU 帧 | ≤ 12.5 ms | ≤ 16.0 ms |
| 固定 Gameplay Tick | ≤ 3.5 ms | ≤ 5.0 ms |
| AI + 导航提交 | ≤ 2.0 ms | ≤ 3.5 ms |
| 主线程流式提交 | ≤ 1.0 ms | 单次不超过 4 ms |
| GPU 帧 | ≤ 13.5 ms | ≤ 16.0 ms |
| 常驻内存 | ≤ 5.0 GiB | ≤ 6.0 GiB |
| 显存 | ≤ 5.0 GiB | ≤ 6.0 GiB |
| 自动保存主线程停顿 | ≤ 2 ms | ≤ 8 ms |
| Cell 穿越可感卡顿 | 0 次 > 50 ms | 0 次 > 100 ms |

P95 在至少 10 分钟压力路线统计。硬门任何一次超出都需解释；GC、Shader 编译和首次导入不允许混入正式游玩。

## 11. 场景预算起点

以下是 F1 起始值，不是永久许可：

- 活跃高逻辑实体 ≤ 80；同时高威胁战斗单位 ≤ 12；群众可使用低频代理。
- 可见渲染实例经实例化/合批后提交目标 ≤ 1800。
- 可见三角形目标 ≤ 5M，阴影投射集合单独预算。
- 实时有阴影灯 ≤ 4 个主要贡献者；装饰灯烘焙或无阴影。
- 全屏透明覆盖避免多层；透明过绘热区在低档必须收敛。
- 活跃骨骼角色按屏幕尺寸分更新率；远处角色使用简化骨骼/动画。
- 一个 Cell 的压缩下载/安装规模和解压内存由 Region Workbench 实时统计。

这些值必须用目标风格实测后调整，不能把“预算尚未最终”理解为无预算。

## 12. 流式与 LOD

Cell 分 Active、Prepared、Visible、Evicting。资产依赖提前清单化，后台 I/O/解压，主线程只提交已准备资源。预取由相机、速度、任务和门触发，提供紧急回退资产。

LOD 不只针对网格：动画更新、AI 感知、粒子、音频、灯光、Shader 和 UI 标记都有距离/重要性档。切换使用滞回，避免边界抖动。地区归位变体优先共享基础资源和差异层。

## 13. 质量档

Low/Medium/High/Ultra 分别控制分辨率缩放、阴影、灯光、反射、粒子、植被、动画距离和后处理。纹理档可独立调整并提示显存。关键玩法可读性不随档位消失。

动态分辨率只处理 GPU 压力，不掩盖 CPU 卡顿。默认策略在 F1 硬件实验后确定；玩家可关闭。

## 14. 大文件与缓存

Git LFS 启用前先确定托管配额、锁策略、缓存、离线备份和客户端最低版本。跟踪源资产和不可再生二进制；可确定重建的巨大缓存不进入 Git。

CI 缓存键包含引擎、导入器、源哈希和平台。缓存不命中只影响速度，不能影响产物。定期验证从空缓存可完整重建。

## 15. 性能工作流

1. 模板创建时选择性能档。
2. 编辑器实时显示局部预算。
3. 每个子地区维护“最坏视角、最密战斗、快速穿越、归位变体”四条路线。
4. CI Headless/自动游玩收集固定硬件指标。
5. 每周趋势比较，不只看单次绿灯。
6. 超预算先定位设计/生命周期/批次，再考虑微优化。
7. 优化后检查画面、玩法和辅助模式回归。

发布前保留每地区独立性能报告和捕获文件。
