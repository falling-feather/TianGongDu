# 18｜安全、依赖与许可证

> 上级文档：[`../01-开发者文档.md`](../01-开发者文档.md)
> 状态：Active Baseline
> 最后更新：2026-07-10

## 1. 保护目标

- 玩家浏览器/设备、账号、Profile、叙事选择与隐私。
- 官方 HTML/JS/WASM、Service Worker、`.tgdpack`、清单和发布身份。
- IndexedDB 本地档、云端 revision、Operation 幂等与冲突分支。
- C++ 源码、源资产、OIDC/数据库/签名密钥和 CI 供应链。

这是单人游戏。防修改/防作弊不能成为侵入式监控的理由；但云同步、唯一奖励和未来在线权益必须验证基本不变量，避免损坏其他设备的档。

## 2. 信任边界

不可信输入：URL/查询参数、DOM/剪贴板、JS↔WASM 消息、网络清单/包、IndexedDB/导入档、Service Worker Cache、OIDC 响应、同步 API payload、模组、DCC 资产、第三方依赖和 CI 产物。

WASM 是沙箱内二进制，不等于内容可信；浏览器同源保护也不能替代鉴权、授权、Schema、大小限制和签名。

## 3. C++/WASM 内存安全

- 外部字节先做大小/版本/哈希，再解析到验证 DTO。
- 使用 span/string_view 必须保证所有权和生命周期；不缓存 JS/临时缓冲指针。
- 整数长度、乘法、偏移和解压尺寸做溢出检查。
- Release 不以 assert 代替输入验证。
- Native ASan/UBSan/模糊测试覆盖同一解析器；WASM 测试验证边界和失败行为。
- JavaScript 只传 ArrayBuffer/受控结构，C ABI 校验长度和枚举。
- 不将 C++ 指针、函数地址或完整对象图暴露给 JS。

## 4. 内容包

- `.tgdpack` 限制压缩/解压大小、文件数、路径长度、图深度和资源尺寸。
- 规范化路径必须留在虚拟根；拒绝绝对路径、`..`、重复/冲突路径。
- 清单、每文件 SHA-256 和 release 签名独立验证。
- 内容不能包含 C++/WASM/JavaScript/Lua、原生库、动态 Shader 源或任意表达式执行。
- 资源类型和 Shader/材质能力白名单；纹理/音频/骨骼防资源耗尽。
- CDN/TLS 保护传输但不替代项目完整性验证。

## 5. Web 安全

- 严格 CSP；尽量禁止 inline/eval，WASM/Worker 所需项精确配置。
- DOM 文本默认 `textContent`/模板转义，不把内容/玩家名作为 HTML。
- OIDC/同步 API CORS 只允许正式 origin；Cookie 时采用 SameSite/Secure/HttpOnly 与 CSRF 防护。
- COOP/COEP 只在 Pthreads 渠道启用并验证所有子资源；隔离策略不能靠放宽任意跨域解决。
- Service Worker scope 最小、脚本短缓存、更新签名/哈希、旧版本安全保留。
- 不信任查询参数决定 debug、包 URL、API origin、账号或构建 variant。
- 多标签用安全锁/广播，只允许一个 Profile 写入者。

## 6. IndexedDB 与本地档

浏览器用户/扩展可修改 IndexedDB，因此加载按不可信处理。校验 SaveVersion、PackageSet、ID、范围、Operation 和哈希；损坏档不覆盖备份。

账号 token/私钥不放普通 Profile。敏感会话优先 HttpOnly Cookie 或平台安全存储；若必须在内存持有，刷新后重新授权。

不使用伪加密宣称保护本地档。加密若用于隐私/云快照，密钥管理和恢复必须真实成立。

## 7. OIDC 与账号

- 使用外部提供方，不存密码。
- 验证 issuer、audience、signature、expiry、state、nonce、PKCE 和 redirect URI。
- 外部 subject 与项目 user 映射稳定，不能以可变邮箱为主键。
- Session 短期、可撤销、轮换；设备撤销和账号退出立即阻止新同步。
- 链接 Guest 前展示本地/云档，不用“登录成功”静默覆盖。
- 账号恢复/删除/导出遵循提供方与项目两层职责。

具体库/提供方进入前做威胁模型和官方流程验证，不手写 JWT/密码学。

## 8. 同步 API

- 每个 Profile 请求检查 user/profile/device 授权。
- request/operation ID 唯一，防重放/重复奖励。
- base revision、Schema/PackageSet 和 payload 版本校验。
- 大小、Operation 数、图复杂度、压缩比、速率和并发限制。
- 服务端不信客户端时间、货币最终值、奖励来源和“已完成任务”布尔值。
- PostgreSQL 事务/唯一约束承担最终去重，不只靠内存缓存。
- 冲突保留双方，不因异常 payload 删除现有 revision。
- 导出 URL 短期、一次性/受限；删除请求可审计并遵守保留政策。

## 9. 服务端与数据库

- Drogon/OpenSSL/PostgreSQL 连接使用 TLS/最小权限；数据库不暴露公网。
- SQL 参数化/ORM 安全 API；动态排序/字段白名单。
- 容器非 root、只读文件系统、受限出网/资源、健康检查。
- Migration 权限与运行时权限分离。
- 备份加密、访问审计、恢复演练；生产数据不复制到开发环境。
- 日志脱敏 token、Cookie、OIDC code、完整快照/Operation 和个人信息。

## 10. 秘密与签名

- 仓库、WASM、JS、Native 客户端没有私有签名密钥、数据库密码和生产 token。
- CI 使用短期最小权限和受保护环境；外部 PR 无发布秘密。
- 内容/release 签名在隔离作业/密钥服务完成，支持 key ID、轮换和撤销。
- 客户端内任何 key 都视为公开；公开验签 key 可以内置，服务秘密不能。
- 秘密扫描覆盖 Git、构建日志、source map、容器和发布包。

## 11. 依赖准入

新增 Axmol/Emscripten/C++/npm/Drogon/PostgreSQL/动画或压缩依赖必须记录：目的、维护状态、许可、版本/commit、哈希、传递依赖、WASM/Native 支持、体积、已知漏洞、替代/移除成本。

- Axmol 使用稳定维护线，Web 补丁单独跟踪上游。
- Emscripten 通过 emsdk/官方镜像固定，不用系统随机包。
- C++ 第三方尽量静态/源码锁定，避免不可审计二进制。
- npm 只用于 Web Shell/工具，lockfile 严格，浏览器产物最小依赖。
- 高风险解析、压缩、网络、密码学、OIDC、数据库驱动需专项审查/模糊测试。

自动更新只能提 PR，不自动合并大版本。

## 12. 漏洞管理

扫描 C/C++、npm、容器、基础镜像和数据库。按可达性、利用条件、玩家/云数据影响分级。Critical/High 可达问题阻断发布；例外有证据、负责人、到期日。

基础平台补丁先在 Preview 回归 Web 单线程/Pthreads、Native、存档和同步，再发布。不能因“最新”在内容锁定期盲升。

建立 `SECURITY.md` 私密报告渠道后对外开放；公开 Issue 不要求披露利用细节。

## 13. 许可证与资产来源

每个代码、字体、音频、纹理、动画 Runtime、模型/参考和 AI 资产记录名称、版本、作者、许可、修改、署名、分发范围和证明。

特别注意：

- Axmol 本体与其三方/扩展许可分别审查。
- Spine 等编辑器/Runtime 不因引擎带扩展就自动获得商用权。
- 字体和音频需允许网页/CDN/原生再分发。
- AI 生成不消除输入参考、商标、人格和版权风险。
- 未授权真实工艺/文化素材不能只因“参考”进入发布包。

构建自动生成 Web/Native 第三方通知和 SBOM。

## 14. 隐私

Guest 本地档不上传。启用云同步时说明身份、Profile、设备随机 ID、版本、进度、保留和删除。遥测另行同意，不能作为同步前提。

禁止默认收集完整 URL 查询、按键、聊天/输入文本、完整日志、广告 ID、硬件序列号和长期浏览器指纹。错误报告由玩家预览类别并选择。

发行市场法规与未成年人要求需专业审阅；本文档不是法律意见。

## 15. 威胁场景门

- 恶意 `.tgdpack` 路径逃逸、解压炸弹、巨大图/纹理被拒绝。
- 篡改 WASM/JS/manifest/package、旧版本重放安全失败/回滚。
- DOM 注入、恶意本地化/角色名不产生 XSS。
- IndexedDB 损坏/修改不执行代码、不覆盖备份。
- OIDC state/nonce/redirect/token 错误拒绝。
- 越权 Profile/设备请求、重复 Operation、并发 claim 只生效一次。
- 数据库恢复后 revision/hash 与对象快照一致。
- 外部 PR 无签名/部署/生产数据权限。
- 模组无法加载代码、联网、覆盖官方 ID 或访问其他 Profile。

这些场景进入 16 的夜间/候选门。
