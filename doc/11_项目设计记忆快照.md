# 《天工渡》项目设计记忆快照

> 版本：0.2 / 交流稿  
> 日期：2026-06-21  
> 用途：给程序、策划、美术和后续协作团队快速对齐当前设计方向。本文不是最终 GDD，而是当前讨论形成的“项目记忆快照”。
> 阅读说明：本文是《天工渡》的独立项目文档，不要求团队预先了解《百工归位》。《百工归位》仅作为旧实验项目和资料参考，文中会把可参考内容拆开说明。

## 1. 一句话定位

《天工渡》是一款开放区域动作 RPG。玩家作为“天工渡人”行走九州，通过结识 NPC、拜师学艺、理解工艺、采集材料、打造器具和完成试炼，逐渐形成自己的战斗门径，并用这些技艺去战斗、探索、解谜与修复失序的世界。

它不是非遗科普游戏，也不是纯模拟经营游戏；非遗器物和传统技艺不是背景皮肤，而是玩家动作、装备、探索、NPC 关系和世界状态的来源。

## 2. 参考原型：《百工归位》

### 2.1 它是什么

《百工归位》是此前制作过的一款非遗生态经营 / 探索实验项目。它不是《天工渡》的前作，也不是团队必须接触过的 IP；它的价值在于已经做过一轮关于“非遗、地区、工艺、材料、NPC、商路、传承风险”的系统化实验。

如果程序团队完全不了解它，只需要先记住一句话：

《百工归位》尝试把中国非遗和手工艺做成一个可运行的经营生态；《天工渡》则要把这批世界观和系统经验转译成开放区域动作 RPG。

### 2.2 可参考的设计遗产

旧项目沉淀了以下内容，可作为《天工渡》的资料库和设计参考：

- 11 个大地区与 43 个小地区的九州行脚框架。
- “传承 / 市场 / 生活 / 精神”四维张力。
- 手艺工序链、材料链、成品、订单、NPC、地区声望、商路稳定、断供危机、结局报告等系统。
- 设计母题：非遗不是放进博物馆就保住，也不是做成爆款就成功，而是在真实生活、市场、传承和精神意义之间不断取舍。

### 2.3 不直接继承的部分

以下内容不建议原样搬入《天工渡》：

- 纯经营结算作为主玩法。
- 大量表格驱动但缺少动作反馈的工艺体验。
- 一开始铺开 11 区 / 43 小地区的完整开发范围。
- 把非遗内容做成百科、图鉴或静态说明。
- 让大地图总览替代真实场景探索。

### 2.4 正式转译原则

《天工渡》可以参考《百工归位》，但必须改变玩家体验重心：

- 旧项目资料可作为世界生态层：地区、产业、人物、材料、订单、商路、社会反馈。
- 《天工渡》是动作体验层：战斗、身法、门径、Boss、探索机关、联机协作。
- 旧项目中的工艺、材料、人物和地区可以转化为动作 RPG 的内容库，但正式实现要重新按动作 RPG 的 Schema 设计。

## 3. 核心设计支柱

### 3.1 一套动作底盘，承载多种门径

为了避免学习成本失控，所有玩家共用同一套基础输入：

- 移动
- 跳跃
- 闪避
- 轻击
- 重击
- 格挡 / 弹反
- 技艺键
- 门径技能键
- 交互 / 采集 / 对话

不同门径改变动作效果、节奏和资源循环，而不是让玩家每学一门技艺就重学一套游戏。

### 3.2 “门径”不是固定职业

当前提到的剑客、伞师、乐师、修真只是初始样例，不是最终只有 4 种职业。项目长期应支持更多古风体系与非遗技艺融合。

“职业”容易让人误以为开局锁死；建议使用：

- 门径
- 流派
- 工法
- 主修 / 辅修

玩家可以主修一种门径，辅修其他门径，最终形成自己的战斗风格。

### 3.3 地区更新驱动长期运营

长期维护不以“堆职业”为主，而以地区内容包为主：

- 一个地区
- 一组 NPC 和师承关系
- 一批材料与产业链
- 一到两条代表技艺
- 一组普通敌人和精英敌人
- 一个核心 Boss
- 若干探索机关
- 一段主线与支线
- 一组装备、技艺、门径扩展

例如：江南纸伞与蓝染、龙泉铸剑、徽州宣纸与徽墨、赣鄱窑火、巴蜀竹器与蜀锦、西域玉作与乐舞等。

### 3.4 战斗终点是“归位”，不是杀戮

为了做到老少皆宜，敌人可以是失序物灵、执念、器障、灾象、机关生命或被污染的风土。战斗可以有强反馈和高操作上限，但表现上不依赖血腥处决。击败 Boss 的结果是修复、净化、唤醒、归位或和解。

### 3.5 NPC 是学习与世界变化的核心

NPC 不只是商店或任务牌。NPC 应承担：

- 师傅：传授门径、技艺、试炼。
- 同行：切磋、协作、竞争。
- 匠人：打造、修复、升级器具。
- 委托人：发布订单和地区事件。
- 见证人：根据玩家行为改变关系、结局与世界反馈。

玩家的成长应尽量通过“遇见人 -> 建立关系 -> 完成试炼 -> 学会技艺 -> 在战斗中验证”来完成。

## 4. 门径体系草案

### 4.1 初始纵切门径

| 门径 | 玩法定位 | 操作特征 | 适合玩家 |
| --- | --- | --- | --- |
| 剑客 | 近战、破招、连段、精准闪避 | 快速接近、看准破绽、短窗口爆发 | 喜欢操作上限的玩家 |
| 伞师 | 格挡、弹反、滑翔、控场、防守反击 | 稳定防守、借风位移、反制远程 | 新手友好，也有高手空间 |
| 乐师 | 节奏、音律、增益、削弱、范围控制 | 按节拍释放、叠加音律、控制敌群 | 喜欢节奏与策略的玩家 |
| 修真 | 内息、符法、身法、蓄势、爆发 | 调息蓄能、术法释放、风险换收益 | 喜欢构筑和幻想感的玩家 |

注意：修真体系需要克制，不能成为“什么都能做”的万能体系。更好的定位是内功和术法底座，可以强化其他门径，但有明确短板。

### 4.2 可扩展门径池

后续可考虑的门径或工法方向：

- 墨工 / 书符：徽墨、宣纸、篆刻、书画，偏封印、改写机关、远程标记。
- 织工 / 牵丝：缂丝、云锦、蓝染、蜀锦，偏牵引、束缚、位移和状态叠加。
- 炉工 / 铸窑：铸剑、制瓷、景泰蓝、金工，偏蓄力、温度、爆发、破甲。
- 影戏 / 幻身：皮影、戏曲、傩面、年画，偏分身、诱导、真伪识别。
- 木作 / 机关：榫卯、竹编、木拱桥、古建修复，偏机关、构筑、地形创造。
- 医工 / 香药：传统医药、制香、药膳，偏恢复、净化、毒、调息和团队辅助。
- 拳械 / 身段：武术、杂技、舞蹈、戏曲身段，偏体术、闪避、空中动作。
- 商路 / 票契：票号、商帮、镖行，偏资源调度、契约、风险转移和世界事件。

这些方向不一定都做成独立职业。它们可以是门径、辅修、技能树、装备词条、探索能力或地区专属玩法。

### 4.3 门径数据结构建议

程序侧应把门径做成数据驱动对象，而不是写死在角色控制器里。

建议字段：

- `id`
- `name`
- `themeTags`
- `baseStance`
- `combatRole`
- `traversalRole`
- `puzzleRole`
- `resourceModel`
- `coreActions`
- `skillNodes`
- `mentorNpcIds`
- `unlockConditions`
- `synergyTags`
- `coOpRole`
- `difficultyRating`
- `accessibilityNotes`

## 5. 核心成长循环

### 5.1 大循环

1. 抵达地区。
2. 认识当地 NPC、匠人、师傅或同行。
3. 了解地区失序事件和产业困境。
4. 探索地图，采集材料，解锁商路或工坊。
5. 完成技艺试炼或工序挑战。
6. 打造 / 修复器具。
7. 获得门径招式、被动、探索能力或装备变化。
8. 用新能力解决战斗、机关、Boss 或社会问题。
9. 改变地区声望、四维状态、NPC 关系和结局记录。
10. 前往下一地区或回访旧地区。

### 5.2 分钟级战斗循环

观察敌人动作 -> 闪避 / 格挡 / 弹反 -> 制造破绽 -> 使用门径技能 -> 积累资源 -> 爆发或控制 -> 调整位置 -> 继续循环。

门径差异体现在资源循环和破绽处理上，而不是完全不同的输入方案。

### 5.3 工艺成长循环

认识师傅 -> 学习工艺阶段 -> 准备材料 -> 选择工法 -> 完成试炼 -> 形成器具特性 -> 战斗验证 -> 返修 / 精进 / 升级。

打造不只是加数值。每件器具应携带：

- 材料来源
- 制作师承
- 工艺阶段记录
- 缺陷或特质
- 外观变化
- 招式变化
- NPC 评价
- 订单或剧情影响

## 6. 地区内容包结构

每个地区更新包建议统一包含：

- 地区名称与地貌主题

- 代表技艺
- 关联门径
- 关键 NPC
- 主要材料链
- 探索能力

- 精英敌人

## 8. 四维系统在《天工渡》中的新用途

旧原型中的四维系统可以作为《天工渡》的地区状态参考，但不再作为纯经营数值，而是转化为地区状态与叙事反馈：

- 传承：师承是否延续，技艺是否失真，古法是否被理解。
- 市场：当地产业是否能生存，商路和订单是否稳定。
- 生活：技艺是否仍与普通人的生活有关。
- 精神：技艺背后的信念、仪式和意义是否还在。

玩家的选择会影响地区四维。例如：

- 过度追求强力装备可能提高市场，损伤传承或精神。
- 只守古法可能提高传承，降低生活与市场。
- 过度娱乐化可能提高生活，稀释精神。
- 尊重师承并完成真实工序，可提升传承与精神，但需要更多时间与资源。

这些状态应影响 NPC 关系、商店、材料价格、支线、敌人失序强度、地区结局与可回访事件。

## 9. 敌人与 Boss 设计方向

敌人不是“文化符号怪物化”，而是失序现象的具象：

- 器障：器物被执念卡住，动作模式来自其工艺缺陷。
- 影魇：记忆、戏影、谣言或误解形成的敌人。
- 灾象：雨、风、火、雾、沙、潮等区域环境失衡。
- 机关生命：古建、木作、榫卯、矿炉等结构失控。
- 执念人形：某个 NPC 或群体困境的投影，击败后回到现实问题。

Boss 应遵循：

1. 一阶段教玩家地区机制。
2. 二阶段要求玩家组合旧能力和新能力。
3. 终局通过修复、归位或选择收束剧情。
4. 不依赖血腥和处决来制造震撼。

## 10. 联机方向

联机应后置，先保证单人体验成立。建议分阶段推进：

1. 单人开放区域动作 RPG。
2. 2-4 人协作挑战区域 Boss。
3. 玩家切磋 / 门径试炼场。
4. 匠会 / 门派 / 共建据点。
5. 区域世界事件与异步订单。

程序设计上从第一天就要避免完全单例化玩家状态：

- 技能、状态、伤害、资源循环尽量数据化。
- 敌人仇恨、Boss 机制、区域事件预留多人输入。
- 技能效果区分本地表现、权威结算和同步事件。
- 乐师、伞师、医工、墨工等要预留团队定位。

但不要一开始做 MMO。先把 1 人玩得好，再扩到小队。

## 11. 技术与内容架构建议

### 11.1 核心原则

- 规则层、内容层、表现层分离。
- 门径、技艺、材料、NPC、敌人、Boss、地区包全部数据驱动。
- 动作手感优先于大世界规模。
- 先做纵切，再做横向扩展。

### 11.2 关键 Schema 建议

最低应建立以下内容表：

- `RegionDef`
- `SubregionDef`
- `MentorNpcDef`
- `PathwayDef`
- `TechniqueDef`
- `CraftProcessDef`
- `MaterialDef`
- `ArtifactDef`
- `EnemyDef`
- `BossDef`
- `QuestDef`
- `DialogueDef`
- `OrderDef`
- `WorldEventDef`
- `RouteDef`
- `ReputationDef`

### 11.3 首个可玩纵切建议

建议第一版 demo 做“江南雨巷 + 纸伞门径”：

- 一个小型江南镇区。
- 一个纸伞师傅 NPC。
- 一条采竹 / 糊纸 / 试伞的轻量工艺链。
- 一套伞师基础动作：格挡、弹反、滑翔、收风。
- 两种小怪：雨魇、破伞影。
- 一个精英怪：断骨伞灵。
- 一个 Boss：雨巷破伞执念。
- 一个旧地图回访点：学会滑翔后到达高处。
- 一个地区四维选择：要不要接商业急单、要不要省略传统工序。

该纵切的目标不是内容多，而是验证：

- 动作手感是否成立。
- 非遗技艺能否自然转成操作。
- NPC 学艺链是否有情感。
- 四维系统是否能进入动作 RPG。
- 地区更新包结构是否可复制。

## 12. 当前明确结论

- 《天工渡》是动作 RPG，不是经营模拟。
- 旧原型中的地区、材料、工艺、四维和 NPC 关系可以作为生态层参考，但《天工渡》必须独立建模。
- 初始门径可以从剑客、伞师、乐师、修真做起，但长期不应限制为 4 种。
- 后续可引入墨工、织工、炉工、影戏、木作、医工、拳械、商路等更多古风体系。
- 玩家成长不是开局选职业，而是大世界拜师学艺、自由主修辅修。
- 地区更新是长期运营主线。
- 联机有潜力，但必须在单人动作体验成立之后推进。
- 第一目标应是高质量纵切，而不是一开始铺完整全国大地图。

## 13. 当前风险

- 范围膨胀：非遗项目极多，必须用地区包和门径框架管住内容。
- 学习成本过高：所有门径必须共享基础输入。
- 修真过强：修真需要定位为一类门径或底层内功，不能吞掉所有体系。
- 文化浅表化：每个技艺必须回答“它怎么玩”，并保留真实工序与师承尊重。
- 联机过早：过早 MMO 化会拖垮动作手感和内容密度。
- 表格堆叠：数据驱动不是堆字段，必须先为首个纵切服务。

## 14. 参考依据

- 中国非物质文化遗产网显示国家级代表性项目按十大门类组织：民间文学、传统音乐、传统舞蹈、传统戏剧、曲艺、传统体育/游艺与杂技、传统美术、传统技艺、传统医药、民俗。该分类可作为《天工渡》长期门径和地区内容池的上层参考。
- 同站截至 2025-03-13 的公开统计显示：国家级代表性项目共 3610 项，国家级代表性传承人共 3998 人。这说明项目长期扩展空间极大，但更需要强 Schema 和范围控制。
- UNESCO 对非物质文化遗产的通行理解强调其是活态实践、知识、技能及相关器物、空间，而非单纯静态文物。该原则支持本项目把非遗转化为“行为、技艺、师承与场景”，而不是只做图鉴收集。
