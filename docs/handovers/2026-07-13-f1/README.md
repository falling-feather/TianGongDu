## 6. 回收与合并顺序

1. 主开发在 02 建立任务、依赖、主责、版本门和 worktree 租约，再向对应长期任务发送 `[ASSIGN]`。
2. 内容、UI、美术分别提交自己的单一意图和证据，不直接合并到集成分支，也不修改根工作区既有 WIP。
3. QA 从不可变候选运行本地矩阵；只在阶段门形成时集中处理 GitHub/三浏览器问题，并以 `[QA]` 回执。
4. 主开发按“合同/内容 → UI/资产 → QA 文档”的依赖顺序集成；任何实现语义冲突回到原 Owner 裁定。
5. 每个角色必须用 `[HANDOFF]` 提供提交 hash、精确标题、变更文件、测试结果、风险、文档和清理状态；“做了很多”或截图数量不能作为完成证明。
