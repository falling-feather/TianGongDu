## 3. 技术来源

下列为本次架构决策直接使用的官方来源，访问日期均为 2026-07-10：

- [Godot 4.7 stable 官方下载页](https://godotengine.org/download/archive/4.7-stable/)：确认 4.7 stable、Windows .NET 构建与导出模板。
- [Godot C#/.NET 官方文档](https://docs.godotengine.org/en/stable/tutorials/scripting/c_sharp/)：确认桌面平台 C# 支持和 .NET 编辑器要求。
- [Godot 运行时文件加载](https://docs.godotengine.org/en/stable/tutorials/io/runtime_file_loading_and_saving.html)：确认可通过 `ProjectSettings.load_resource_pack` 加载 PCK/ZIP 数据包。
- [Godot PCKPacker](https://docs.godotengine.org/en/stable/classes/class_pckpacker.html)：确认运行时资源包的创建与加载能力。
- [Godot 多分辨率](https://docs.godotengine.org/en/stable/tutorials/rendering/multiple_resolutions.html)：用于分辨率与缩放策略。
- [.NET 官方支持策略](https://dotnet.microsoft.com/en-us/platform/support/policy)：确认 .NET 10 为当前 Active LTS；最终 Godot 目标框架仍需 F1 兼容实验锁定。
- [JSON Schema Draft 2020-12](https://json-schema.org/draft/2020-12)：内容契约规范源。
- [Git LFS 官方站点](https://git-lfs.com/)：大二进制资产版本管理机制与当前安全更新提示。

技术文档只引用官方文档、标准或项目原始仓库作为事实依据。博客和二手文章可用于发现问题，但不能单独成为长期决策证据。

## 6. 研究登记格式

每项真实工艺研究记录：主题、来源类型、作者/机构、标题、日期、链接/馆藏、访问日期、可验证事实、团队推断、幻想改写、使用位置、审阅人和许可状态。

资料可信度排序：行业标准/博物馆/学术出版/从业者访谈 → 高质量专著与机构文章 → 多源互证的一般资料 → 仅供灵感的二手内容。生成式 AI 输出不能作为事实来源。

## 8. 变更检查

提交文档前回答：

1. 信息是否已经有唯一归属？
2. 是未来计划还是已完成事实？
3. 是否改变世界正史、产品边界或长期架构？
4. 是否需要同步 Schema、目录 JSON、测试或迁移？
5. 是否留下来源、负责人、日期和验证方式？

如果答案不清楚，先修归属和证据，不新增“临时文档”。
