# 04｜游戏设计总纲

> 状态：Active
> 职责：定义玩家循环、战斗、武器、探索、关系、任务、工艺、成长与经济
> 最后更新：2026-07-14
> 维护者角色：游戏总监；战斗、关卡、系统和叙事负责人按章节会签

## 1. 目标体验

《天工渡》的核心不是“打怪拿更大的数字”，而是逐渐学会看见一个地区如何运转。玩家刚到一地时只看见器物与麻烦；熟悉之后看见工序、人情和利益；完成归位时，能理解为什么同一件事没有简单的好坏答案。

四个体验动词：

- **行**：沿街巷、水路、山道和工场探索，利用天气、地形和器具改变路线。
- **问**：与有日程和立场的人交往，收集并质疑口述、账页、工痕和环境证据。
- **试**：用兵击、工灯和工艺操作验证判断，承担失败与资源代价。
- **归**：在 Boss、工程或公共裁定中恢复用途，并决定怎样记录责任与功劳。

## 2. 时间尺度循环

### 2.1 30 秒战斗循环

观察攻击前摇与场地 → 守势/闪身/位移 → 用武器架势制造破绽 → 工灯照见错位 → 布印改变规则 → 完成处决或归位窗口。

### 2.2 5 分钟兴趣点循环

发现地标或异常 → 与环境/NPC 建立问题 → 在探索、交涉、工艺或战斗中处理 → 获得新线索、关系、材料、捷径或地区状态。

### 2.3 通常 30–70 分钟的子地区循环

进入并建立安全点 → 认识当地日常 → 学会独有探索语法 → 处理一条主线和若干可选内容 → 改变空间/日程 → 解锁回环与下一地区。

地区终战型子地区可达 90–100 分钟，但必须拆成 20–35 分钟可保存、可退出的段落；总时长不能是一场无法中断的连续关卡。

### 2.4 5–10 小时地区循环

陌生来客 → 得到有限信任 → 进入生产深处 → 发现表象与根因冲突 → 多方协作或裁定 → 地区 Boss 归位 → 回看居民、市场、路线和记录如何变化。

## 3. 战斗共同底层

### 3.0 2.5D 斜向全景空间与相机

正式玩法采用受控俯斜镜头下的 2.5D 斜向全景空间。玩家在地面平面上做相机相对的八方向移动，横向与纵深共同承担接近、拉开、包夹、绕背、躲避地面危险和争夺工位；独立高度只承担跳跃/纵跃、击飞、落差、桥层和明确的环境能力，不把屏幕竖直方向误当作世界纵深。战斗空间必须让地面轮廓、可走边界、遮挡关系和敌我脚点始终可判断。

《斗战神》是第一指导，用于校准斜向全景构图、角色与场景尺度、空间层次、东方材质重量、群战调度和镜头场面感；《暖雪》只作为第二参考，用于校准俯斜视角下的轮廓、读招和输入简洁度。只借鉴这些高层设计语言，不复刻其角色、地图、UI、动作或资产。

相机以固定方位或关卡作者控制的有限方位为基线，自动跟随并提供移动/攻击方向前瞻；Boss、公共工程与狭窄室内可进入受控竞技场框或平滑切镜。玩家可调镜头距离、前瞻、抖动和遮挡透明度，但不需要持续拖动鼠标旋转视角。关卡以“全景 Cell + 短连接 + 回环捷径 + 枢纽”组织，并同时设计前景遮挡、中景战斗面和远景地标。

战斗固定 60 Hz 模拟。角色核心资源为：

| 资源 | 作用 | 恢复与风险 |
| --- | --- | --- |
| 体力 | 移动、防御和武器动作成本 | 停止高强度动作后恢复；耗尽会出现喘息窗口 |
| 定力 | 抵抗硬直并维持架势 | 精确防御、合理站位恢复；被连续压制会破势 |
| 工灯势 | 使用工灯、工印与协作技 | 观察正确工痕、精确格挡、环境交互和关系协作获得 |
| 证据槽 | 当前敌人/工程的可验证错位 | 必须通过具体行为发现，不能靠普攻填满 |

伤害只解决“能否继续抵抗”，失序值解决“是否暴露结构”，证据解决“是否理解根因”。普通敌人可直接制伏；精英与 Boss 的完整奖励和世界变化要求归位。

### 3.1 输入原则

- 轻击、重击、守势、闪身、武器技、工灯、交互和快捷物品构成基础输入，不为每个新武器增加整套键位。
- 移动输入先归一为地面平面二维向量，再按受控相机方位换算为世界方向；镜头切换不得偷改已提交的规则位移。
- 输入缓冲、取消窗口和无敌帧均由数据定义，但必须在统一范围内，避免内容作者制造不可读特例。
- 键鼠与手柄同等优先；锁定不是强制，软锁定和镜头辅助可以独立调节。
- 战斗 UI 提供读招和状态，不替代世界中的动画、声音和材质反馈。

## 4. 战斗系统 A：护牒兵击

“护牒”代表玩家以天工院通行牒承担介入责任；“兵击”代表所有武器共享的动作语法。

### 4.1 五段循环

1. **识招**：从动作、声音、环境征兆判断攻击与用途。
2. **守身**：格挡、偏转、闪身或借器位移；没有万能选项。
3. **夺位**：攻击弱侧、破坏支撑或把目标引入有利工位。
4. **破势**：削减定力、打断错误工序，使失序结构显露。
5. **择终**：直接制伏、保留观察、触发工灯归位或与 NPC 协作。

### 4.2 设计约束

- 普通战斗 20–60 秒；精英 90–180 秒；章节 Boss 4–8 分钟；地区 Boss 8–15 分钟。
- 每类敌人至少有一个位置问题、一个节奏问题和一个工灯交点。
- 难度提升优先增加组合和决策压力，不优先增加生命值。
- 群战同时高威胁单位原则上不超过 3 个；其余单位承担空间、辅助或节奏角色。
- 处决不必致死，可表现为卸力、拆解、封存、驱离或使器物停机。

## 5. 战斗系统 B：工灯定印

工灯不是扫描键，也不是暂停后的选择题。它是一套与实时位置、时机、资源和证据绑定的战斗系统。

### 5.1 三种工灯动作

- **照见**：短时聚焦一个工痕、连接点或行为，风险是视野和移动受限；正确观察生成证据。
- **引序**：把敌人、机关、流体、热量或声音导向新的工序节点，改变场地规则。
- **定印**：消耗工灯势，在有足够证据时写入临时规则，例如导流、退火、校轴、卸荷、固色、复文。

### 5.2 归位裁定

Boss 进入归位窗口后，玩家必须根据已收集证据选择处理方案。方案不做纯道德二选一，而在用途、责任、风险与人情之间取舍。完整结果由四部分组成：

- 器物/工程怎样恢复或停用。
- 谁承担维修、赔偿或传承责任。
- 谁的劳动被写入《天工录》。
- 地区的路线、价格、日程、关系和后续任务怎样变化。

证据不足仍可结束战斗，但会得到保守或代价更高的归位，且不能解锁全部关系与工艺成果。

### 5.3 两系统交叉

| 护牒兵击行为 | 工灯收益 | 工灯反哺 |
| --- | --- | --- |
| 精确格挡 | 显露受力方向 | 校轴印延长反击窗 |
| 破坏支撑 | 暴露错误连接 | 卸荷印改变倒塌路线 |
| 引敌入场 | 验证环境假设 | 导流/退火印生成新站位 |
| 保留处决 | 延长观察机会 | 归位后获得协作而非单纯掉落 |

## 6. 武器体系 A：伞器

伞器是江南主武器，强调保护、控制、借势和天气。它由伞骨、纸面、桐油与铰轴四个工艺域组成。

### 6.1 三架势

- **檐守**：展开伞面，降低正面压力并强化精确偏转；移动较慢，怕绕后与持续重压。
- **翻花**：半开快速旋击，控制近身和飞行物，适合积累失序；单次伤害较低。
- **风渡**：借风、坡面或冲击位移，进行突进、撤离和空中衔接；消耗体力高。

### 6.2 成长

成长不是线性换更大数值，而是“工艺部件 + 招式谱 + 师承选择”：

- 伞骨决定回弹、守势和形变能力。
- 纸面决定元素/天气互动与可修复性。
- 桐油决定防水、燃烧和附着效果。
- 铰轴决定开合速度、架势切换和机关联动。

终极能力“四时合伞”要求玩家在一场战斗中正确完成四种环境应对，而非按下冷却大招。

## 7. 武器体系 B：剑器

剑器在龙泉完整展开，强调用途、材料权衡、距离判断和精确反击。选矿、叠锻、淬火、研磨四个工艺域直接改变操作手感和招式窗口。

### 7.1 三架势

- **试锋**：短前摇、快速确认距离与敌方硬直，适合探索招式，但定力收益低。
- **藏锋**：以守待攻，储存锋势并强化精确反击；主动追击能力弱。
- **回炉**：暂时降低输出，把热量/失序转化为韧性、属性或破甲调整；需要安全工位。

### 7.2 工艺权衡

高硬度提高锋利却更易断纹；高韧性容错更高但破势慢；极端淬火强化一种环境却削弱另一种。玩家可以在安全工坊永久调整，也可以在特定场景用“回炉”做有限临时调校。

终极能力“百炼归一”让玩家把本场观察到的敌方材料与受力证据转为一次正确用途的剑式，不能在未知目标上发挥满效。

## 8. 未来武器预留

1.0 只完整发布伞器和剑器。架构预留墨尺、机弩、绳结器、药杵等未来体系，但不得为了假想武器抽象到无法实现当前内容。未来武器必须复用共同输入、资源、命令、效果和工灯接口，同时拥有独立工艺域和地区用途。

## 9. 探索与地区状态

### 9.1 地图不是清单

地图从陌生轮廓开始，通过亲自到达、NPC 指路、测绘、工灯观察和公共工程逐步变清楚。兴趣点在发现前不以问号铺满；线索可以给方向、声音、地标或时间窗。

斜向全景探索使用纵深岔路、高低层、可破坏/可修复通路、绳索/水流/风势、前景遮挡入口和地区工艺机关制造差异。纵跃、落差与桥层服务路线和战斗，不以高惩罚精密平台跳跃拖慢叙事；坠落通常回到最近安全落点并扣除有限资源。

### 9.2 天气与日程

天气改变路线、敌人、材料和 NPC 行为，但主线不得被纯随机长期阻断。每个地区有可预测的日程和天气窗口，旅店、公告、观察和关系能力可以获知或调整。

### 9.3 归位前后

每个大型子地区至少有三类可见变化：空间/设施、NPC 日程/对白、经济/任务。地区归位不是播放片尾后复原世界，而是提供一段可游玩的后日常。

## 10. NPC 与关系系统

核心 NPC 由五个通用维度组成：信任、认同、亏欠、熟悉、公开声誉。UI 不必暴露全部数值，但系统要能表达“相信你的能力但不同意你的判断”这类关系。

每个核心 NPC 至少具备：

- 工作、休息、社交和异常四类日程状态。
- 三阶段关系弧，不要求都变成亲密关系。
- 两类以上可玩交互，例如指导、交易、同行、辩契、工艺、决斗、研究、公共工程。
- 一条成链任务，以及对至少一个地区归位结果的反应。
- 忙碌、缺席、受伤、迁移等状态下的替代交互，不因站位变化让任务死锁。

关系奖励优先是新方法、新信息、新协作和新责任，其次才是数值加成。

## 11. 任务与选择

任务由目标图、条件、事件、结果和恢复策略构成，不由一长串场景脚本构成。

选择的有效反馈至少命中两项：

- 立即反馈：对白、表演、资源、战斗或路线。
- 中期反馈：后续任务、价格、NPC 日程、协作能力。
- 地区反馈：公共设施、声誉、归位方式、结局段落。
- 记录反馈：《天工录》词条的署名、措辞、证据和异议。

任务允许暂时失败和替代路径，但不能因为玩家错过一个时间窗永久破坏主线。真正不可逆选择必须在确认前说明影响层级，并自动保存可回退节点。

## 12. 工艺、物品与经济

物品分为材料、工具、器件、消耗品、记录与纪念物。装备没有无限颜色稀有度阶梯；价值来自用途、工艺组合、来源和可维护性。

工艺循环：观察需求 → 获取/交换材料 → 选择工序与权衡 → 试用 → 维修/重调 → 留下谱系。失败产生可利用的次品、信息或返工，不简单吞掉全部材料。

经济以地区供需、关系和公共工程为主。玩家不能靠无风险无限买卖击穿经济；价格变化有上下限和恢复曲线。关键剧情物品不占普通负重，常用材料提供分类收纳与批量处理。

## 13. 成长结构

成长分四条并行轨道：

1. **渡人能力**：工灯、证据、协作和地区通行。
2. **武器师承**：伞器/剑器的架势、招式和工艺部件。
3. **百工见闻**：通过实际使用解锁工艺理解与跨地区组合。
4. **人物关系**：导师、伙伴、商路和公共支持。

没有无限等级碾压。数值成长控制在可读范围，后期深度主要来自组合、资源调度、敌人变式和玩家知识。洗点在安全点低成本提供；不会用不可逆构筑惩罚实验。

## 14. 失败、难度与辅助

失败后保留已发现的基础线索和无争议记录，回滚战斗消耗与未裁定选择。Boss 重试点位于演出和长路之后，不重复已看过的不可交互段落。

难度由可独立调节的维度组成：敌人进攻性、招式提示、精确窗口、资源宽容、导航提示、工灯时间压力。辅助模式可延长窗口、降低镜头冲击、自动完成高频按压，并保留内容和归位结局；不以羞辱性标签区分玩家。

## 15. 内容节奏红线

- 连续两场强制战斗之间应有探索、关系、工艺或叙事缓冲。
- 同一敌群组合在 30 分钟内不原样重复第三次。
- 新机制出现后 10 分钟内要有安全教学、正式应用和一次变式。
- 主线不可连续 10 分钟只听对白；长对话提供移动、观察、选择或章节化回看。
- 支线不能只是从甲处跑到乙处交物；至少改变信息、方法、关系或空间之一。
- 每地区前 30 分钟必须展示当地日常，而不是立刻用灾难清空居民。

## 16. 1.0 内容映射

江南完整教授伞器、基础护牒兵击与工灯定印；龙泉用剑器和锻造权衡证明框架可扩展；徽州强化关系、契约、非战斗裁定与跨区后果。精确子地区、Boss 与 NPC 见 [`07-1.0地区与内容蓝图.md`](07-1.0地区与内容蓝图.md)。

## 17. 玩家本体、推进与装备边界

玩家角色是“固定职业与成长弧、可定制称谓和外观”的半预设主角，正史边界见 05。基础动作全集在所有武器间复用：站立/八向转向、地面平面行走/奔跑、短可变高度纵跃、跨越指定障碍、落下恢复、守势、闪身、轻击、重击、武器技、工灯、交互和快捷物品。自由飞行、蹲伏潜行、精密墙跳和自由攀爬不属于 1.0 通用动作；若局部关卡需要，作为明确的环境能力而非隐藏基础操作。

首次流程按江南 → 龙泉 → 徽州推进，以保证战斗、工灯和记录系统逐步教学。完成每地关键章节后可自由回访；回访内容不能要求玩家重走整条主线。江南期间主武器固定为伞器，龙泉获得剑器后解锁两个装备槽：

- 战斗中一次只有一个武器体系处于 Active；另一把保留部件和冷却状态。
- 仅在自由移动、未受击、未处于归位/交互时切换，F1 之后以 30 Tick 左右的可调过渡作为起始值。
- 切换不清除体力、定力、工灯势、效果或敌人仇恨；不能靠来回切换刷新资源。
- 安全点可改部件、招式谱和快捷物品；战斗内只允许已装备的有限调校。
- 没获得剑器时，所有双器 UI、掉落和提示保持隐藏，不用空槽提前打扰玩家。

战败不等同于主角正史死亡。普通战斗表现为受伤、被迫撤回或护牒自动封存现场；Boss 失败恢复到战前安全快照。人类敌对者默认制伏、缴械、驱离或停止作业，1.0 不以处决人类作为奖励循环。

```text
Boot → Title/Profile → Loading → InWorld
                              ├─ Exploration
                              ├─ Combat
                              ├─ Dialogue/Interaction
                              ├─ DeepMenu (paused)
                              ├─ MajorChoice (paused)
                              └─ Defeat/ChapterResult (paused)
```

模式由 GameCore 的 `SessionMode` 与 Web Shell 的 `UiMode` 协商，但只有 GameCore 能决定战斗、任务和暂停语义。DOM 面板“看起来关闭”不等于恢复 Tick；必须收到已验证的关闭命令。输入先映射为 Action，再按上下文解释：

| Action 组 | Exploration | Combat | Dialogue/Menu |
| --- | --- | --- | --- |
| Move/Traverse | 地面八向移动、纵跃/跨越 | 保持可用，受动作状态约束 | 焦点导航或停用 |
| Light/Heavy/Guard/Evade | 可做预备动作，不打无敌 NPC | 权威战斗命令 | 确认/返回只用独立 UI Action，不复用攻击副作用 |
| Lantern/Interact | 调查、照见、交谈 | 工灯/情境协作 | 提交明确选择 |
| Pause/Journal/Settings | 打开深层面板并暂停 | 除不可暂停演示测试外均可暂停 | 返回上一层，焦点可恢复 |

模态、文本输入或按键重映射开始后，Web Shell 发出 `InputContextChanged` 并清空持续动作；Gameplay 不接收浏览器 key repeat。手柄、键鼠和触控同时存在时，最近一次有效设备只改变提示图标，不改变 Action 含义。

## 19. 战斗结算管线

每次动作按同一顺序解析，避免武器、Boss 和表现各写一套规则：

1. 校验 Session/角色状态、输入缓冲、资源和目标。
2. 锁定动作实例、成本和取消窗；资源在明确的 `commitTick` 扣除。
3. 执行规则位移与权威碰撞查询；动画 Root Motion 只跟随结果。
4. 生成命中候选并按稳定键去重、排序。
5. 先处理无敌/偏转/格挡，再处理生命、定力、失序和效果。
6. 工灯读取已经成立的受力/材料/工序事实，生成或反驳证据。
7. 破势、倒地、阶段变化和归位窗口在 Tick 末提交。
8. Presentation 接收命中停顿、音效、相机和 VFX 事件；它不能反向补判命中。

生命归零解决“是否停止抵抗”；定力归零产生破势；失序达到条件暴露结构；证据达到组合条件开放工印/归位。四者禁止折叠为同一条 Boss 血条。

## 21. 敌对实体与重复挑战的世界规则

| 类别 | 来源 | 战斗终点 | 是否可重复出现 |
| --- | --- | --- | --- |
| 失序器物/工偶 | 工序或用途冲突使器物持续错误执行 | 停机、拆解、校正或归位 | 根因未处理前可由世界事件再生；归位后转为稀少变体/训练 |
| 环境祟影 | 水位、声纹、热量、契约等留下的局部回声 | 驱散并修复触发条件 | 天气/日程条件下可重现，但不重复发唯一剧情奖励 |
| 人类阻碍者 | 误解、利益、护运、比试或违法作业 | 制伏、说服、缴械、撤离 | 仅在合理巡护/比试/治安事件中；不把居民当无限刷怪 |
| Boss/大型祟局 | 多层工事和社会冲突的集中失序 | 证据裁定与归位 | 正史只结算一次；之后在天工录/试炼中进行非正史重演 |

普通玩家无需天生看见所有祟：多数人能感知异常结果，工匠能认出本行业征兆，执灯渡人通过工灯看见可验证工痕。地区归位后仍保留少量训练、未完全修复的边缘回声和新日程事件，但敌群密度必须显著下降或改变行为，让玩家感知“修复真的发生”。

## 23. 1.0 成长合同

1.0 不设可无限刷取的角色等级。玩家能力由可追溯来源解锁：

| 轨道 | 1.0 目标宽度 | 主要来源 | 禁止 |
| --- | --- | --- | --- |
| 护牒/渡人 | 12 个左右里程碑节点，含生命/体力小幅上限、证据/协作能力 | 主章、地区归位、天工院复核 | 靠刷怪等级碾压主线 |
| 工灯 | 照见/引序/定印基础 + 每地区 2–3 个用途工印 | Boss 证据、公共工程、研究 | 扫描即给答案、无限免费定印 |
| 伞器 | 3 架势、12–18 招、每个工艺域 3 个左右可选部件、1 宗师能力 | 江南教学/导师/支线/试伞 | 部件只有攻击力高低 |
| 剑器 | 3 架势、12–18 招、每个工艺域 3 个左右可选部件、1 宗师能力 | 龙泉教学/试剑/锻造/裁定 | 后获得便完全替代伞器 |
| 百工见闻 | 约 30 个可组合知识标签 | 实际观察、试验、工艺任务 | 只读图鉴自动加永久百分比 |
| 关系协作 | 24 人各 3 阶关系弧；每人至少一个专业/协作结果 | 共同经历、价值选择、承担责任 | 重复送礼刷满所有人 |

上限成长控制为“降低挫败、扩大选择”，不让后期普通攻击跨越整套机制。生命/体力等永久数值相对开场增长的 1.0 起始上限不超过约 50%，最终比例需由 M1 敌人/辅助盲测锁定。洗点在安全点低成本完成；剧情获得的知识/关系不会因洗点丢失。

武器教学顺序：江南 `jn_chapter_02` 解锁伞器檐守/翻花，`jn_chapter_03` 前后获得风渡，后续三章逐步开放工艺部件与交叉招，`jn_chapter_07` 完成宗师路线的首个裁定；龙泉以同样节奏展开剑器。F1 只实现伞器两架势和必要动作，不把纵切招式量冒充完整武器体系。

## 24. 经济与物品制作预算

正式名称/数量在 OQ-114 与 M1 经济 Greybox 锁定；开工使用以下上限级别，防止系统无界膨胀：

- 一个通用日常货币账本，不预设付费货币；行券、关系担保和公共工时是条件/权益，不伪装成第二套万能货币。
- 每地区约 18–24 个有明确来源/用途的核心材料；跨区通用材料另设小集合，目标总量 60–80，而非数百换色矿石。
- 每武器约 12 个功能部件（四工艺域各约 3 个方向）和少量宗师部件；同域只装备一个，替换保留旧部件。
- 1.0 约 15–20 种常用消耗品/现场工具，覆盖恢复、抗环境、调查和路线用途；快捷栏 3 格。
- 剧情/证据/护牒/记录进入独立关键物品层，不占普通背包，不可卖出或意外销毁。
- 材料按类别堆叠、无传统负重惩罚；超额提供仓库/自动整理，不能逼玩家在任务途中丢工艺证据。
- 每地区 2–3 个核心商店/工坊和若干日程摊位；商品由地区状态、关系和供需改变，但主线必需品始终有恢复路径。
- 重复敌人主要掉维修/通用材料，不掉无限随机稀有装备。关键部件来自导师、工艺、Boss 归位、公共工程和探索。

价格曲线有地区基价、供需修正、关系/契约修正和上下限。买卖同一物品的往返价差、库存刷新和任务奖励必须通过经济模拟，避免无风险套利；单机 Profile 可被玩家修改，因此普通经济由本地玩法负责，不把云端当半权威反作弊。

## 25. 奖励优先级

高价值奖励按以下顺序设计：新方法/路线 → 新协作/关系责任 → 新工艺部件/招式 → 世界/记录变化 → 有用途的资源 → 纯数值。每个主章至少交付前四类中的一项；Boss 不允许只掉“更高品质同款武器”。

唯一奖励携带 `rewardDedupKey`，失败重试、刷新和同步重放不重复发放。证据不足路线也给足以继续主线的基础奖励；完整归位奖励更多理解、关系和组合能力，不用永久数值惩罚逼玩家查攻略选“正确答案”。

## 26. F1 前两 Beat 玩家动词与首玩节奏

本节是 `F1-GAME-01-CONTENT-A` 的第一阶段玩法合同。目标样本是未看开发界面、未接受口头提示的首玩玩家；从获得控制到完成沈砚训练，真人中位数目标为 15–17 分钟。下表只定义预期节奏与可观察行为，不构成时长验收。`targetMinutes`、Objective 数量、自动回放 Tick 和策划估时都不能替代真人盲测。

### 26.1 逐分钟动词与 30–90 秒变化

每一行必须在 30–90 秒内至少改变一项：判断、空间、风险或反馈。玩家熟练到连续两行都没有新信息时，内容评审应压缩或重写，而不是强制慢走。

| --- | --- | --- | --- | --- |

| 00:45–01:35 | 在高水痕、溺水货单、循钟声之间选择 | 两条可选调查线与一条明确跳过线分居横向空间 | 选择被确认；未选线索不再推进本次流程 | 选择与空间比较 |

### 26.2 两种架势的完成语义

- 檐守式用于正面受力情境。必须在已激活训练威胁中完成一次有效格挡、一次檐守重击，并击破 Actor 107（archetype `f1_training_eavesguard_target`）；仅切换架势或空按动作不得推进。
- 翻花式用于斜向越线情境。必须切换架势、穿过训练线、完成一次有效闪避反制、一次轻击、一次重击，并击破 Actor 109（archetype `f1_training_flower_turn_target`）；绕开场地直奔沈砚不得推进。
- 两个耐久训练伞架承担可重复来击与失败恢复，两个证明靶承担阶段清除。它们是通用 Actor、Combat Trigger 和 Combat Outcome 的组合，不是 Objective 专属 Runtime。

### 26.3 顺序、跳过、失败与返回矩阵

| 情境 | 明确行为 | 禁止行为 |
| --- | --- | --- |
| 可选线索跳过 | 选择“循钟声”后由主水尺完成同一主 Objective；高水痕与货单不再阻塞，且三路均不发唯一奖励 | 隐藏式强制全收集、跳过后永久少关键能力 |
| 错序交互 | 前置 Objective 未完成时拒绝推进并给出就地原因；状态、奖励和选择均不变 | 悄悄吞输入、提前完成后续 Objective、重置整段 |
| 重复交互 | 已接受的交互再次进入只回显完成状态；选择不改写，Objective/奖励不重复提交 | 重复推进、刷奖励、借刷新切换互斥选项 |
| 工艺错误 | 过载快结路线把纠正动作作为所选方案的后续，不判任务失败；钟码错序只反馈并允许立即重试 | 全局回档、等待计时器、强制重走渡船 |
| 训练受击/倒地 | 保留已提交 Objective 和所选训练线；从 `f1_safe_point_shen_yan_training` 恢复，按完成边界重建应激活 Actor，未完成动作重新证明 | 把失败尝试计入合格时长、重复发奖、敌人无限复制 |
| 离开后返回 | 根据已选线与已完成 Objective 重建当前标记、休眠/激活 Actor；已击破证明靶保持已解决语义 | 因 Presentation 对象消失而丢进度、重新播放并推进对话 |
| 完成 Beat 后返回 | 雨渡线索与训练设施可提供无推进回看/练习；主线 Beat 不回退 | 再次提交安全点、再次解锁、再次发奖 |
| 安全点 | Beat 1 完成后在门楼提交；Beat 2 失败恢复使用雨渡训练场安全点；恢复必须可由 Quest 状态确定性重放 | 依赖加载时间、动画回调或本地面板状态决定恢复结果 |

盲测报告必须把总墙钟拆成合格时间、暂停/设置、失败尝试、空跑/迷路、重复行为五列。正常首次观察、选择、对话、工艺推理和首次成功战斗计入；挂机、加载、故意绕圈、失败重试和已解决敌人的重复刷取不得用于满足 15–17 分钟目标。

### 26.4 玩家反馈投射边界

Schema 1.5.0 曾为前两 Beat 增加八个 Definition-owned Quest UI cue；Schema 1.6.0 在同一 cue 下补齐 14 条精确 `attemptEvidenceRules`，以 source、焦点 Objective、主/次结果 ID、状态和拒绝原因派生合格时间分类。Runtime signal 不再携带或覆盖分类；缺失、重复或不兼容规则均失败关闭。cue 仍只把权威 Quest/Combat 快照路由成选择、交互反馈、阶段、动作证明和恢复模型，不新增 Objective、不提交进度，也不凭 UI 事件贡献额外分钟。快速结被系统接受后仍可投射为负向受力反馈；训练重击被接受但证明靶结果被拒绝时，面板必须同时保留“动作已触发”和“目标证明未成立”，不能把二者压成一次成功。

`tests/fixtures/f1-first-two-beats-ui-events.json` 的 14 条事件是覆盖八个 cue 的独立投射目录，不是一条可按 sequence 直接播放的合法任务路线。正式生产者必须从每个事件对应的独立 Quest/Combat 快照重建模型；不得用夹具顺序自动推进、累计时长或替代两条固定内容回放。

通用 producer 修复 `87a9771` 已以 `feb6f88` 接入本分支；随后主开发在 `a4fe0b97` 发布通用证据规则合同，本分支以 `cbb1223` 接入公共 API，并由内容提交 `3eb2192` 把唯一 Definition 升至 1.6.0。真实 F1 Definition 在当前 Beat 同时暴露 11/14 个 stage-active Objective 时，投射仍只聚焦本次 signal，不压扁同 Beat 其他 Objective；跨 Beat Active、locked focus、未知/歧义证据和 stale intent 继续失败关闭。两条固定内容回放各自从真实 Session 生成三次 choice 投影并校验完整选择意图，冻结的 UI 校验值仍为 `955f3abb832aa33f` 与 `45f516efd2d8c92d`。这证明 Definition→producer→固定回放的分类权威与确定性，不证明其余交互/战斗/恢复事件已有正式 emitter/renderer，也不证明 15–17 分钟真人体验。
