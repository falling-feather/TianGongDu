# 08｜UI、UX 与可访问性

> 状态：Active
> 职责：定义玩家信息层级、输入、反馈、菜单、分辨率与辅助功能
> 最后更新：2026-07-11
> 维护者角色：UI/UX 负责人；Web、Gameplay、可访问性与 QA 共同会签

## 1. 设计原则

1. **先看世界，再看仪表**：关键危险必须由动作、声音和场景表达，HUD 用于确认而非替代。
2. **只显示当下能行动的信息**：不把任务、Buff、材料、地图和教程同时堆在常驻层。
3. **工艺可读，不做表格考试**：复杂工序通过空间、形变、颜色、声音和逐级提示表达。
4. **输入等价**：键鼠与手柄都有完整路径；所有关键操作可重映射。
5. **辅助不删内容**：降低操作压力不应剥夺 Boss 归位、剧情或奖励。
6. **面板不能偷偷运行危险世界**：深层菜单默认暂停单机模拟；明确例外需 UI 提示。
7. **Canvas 与 DOM 各司其职**：Axmol/Canvas 只负责世界、战斗和必须与相机同步的标记；文字密集 HUD、菜单、设置和辅助控件优先使用语义化 DOM。

## 2. 信息层级

| 层级 | 名称 | 示例 | 是否暂停 |
| --- | --- | --- | --- |
| L0 | 常驻战斗 HUD | 生命、体力/定力、工灯势、当前架势 | 否 |
| L1 | 情境层 | 交互提示、拾取、短教程、证据发现、方向线索 | 否 |
| L2 | 快速层 | 武器轮盘、快捷物品、同伴命令、简易证据选择 | 时间减缓或否 |
| L3 | 深层系统 | 地图、行记、天工录、背包、关系、工艺 | 是 |
| L4 | 模态层 | 重大选择、失败/重试、章节结算、设置 | 是 |
| D0 | 开发层 | 实体检查、任务图、性能 HUD、内容编辑器 | 仅开发构建 |

D0 与正式 UI 使用独立入口和主题，Release 构建默认剔除；不能让调试按钮混进普通玩家路径。

## 3. 浏览器视口与 1920×1080 基准布局

HUD 采用边缘锚定和 5% 安全区，不把核心信息压在角色附近：

- 左下：玩家生命、体力/定力、状态异常；默认紧凑，可拆分资源条。
- 右下：武器架势、工灯势、快捷物品和输入提示。
- 顶部中央：Boss/公共工程状态，仅在相关遭遇出现。
- 左侧中部：最多 3 条短目标或现场任务状态，可一键收起。
- 右侧中部：证据/工印发现的短暂堆栈，自动归档到天工录。
- 世界中心附近：只出现短时交互、精确格挡和归位窗口，不长期覆盖角色。

DOM HUD 覆盖在 Canvas 上方，但 `pointer-events` 只在真实交互区域启用，避免吞掉战斗输入。Canvas 使用内部逻辑分辨率与设备像素比缩放，DOM 使用 CSS 像素和可独立缩放的根变量；两者通过稳定世界坐标投影桥对齐，不共享布局对象。

16:10 和 21:9 通过锚点扩展视野，HUD 保留在可配置安全框内；超宽不能展示本应被遮挡的战斗生成区。窗口过窄时进入“窄屏提示/触控布局”，不能把桌面 HUD 等比缩到不可读。浏览器缩放 80%–200%、全屏切换、地址栏变化和 DPR 改变必须重新布局但不重建游戏 Session。

## 4. 战斗 HUD

### 4.1 玩家状态

生命使用形状、数值和颜色多重表达；体力与定力不能只靠红绿区分。资源条低于阈值时有一次清楚提示，避免持续闪烁。

架势图标显示当前形态和切换方向。招式冷却只显示主动且近期需要的信息，不在屏幕铺满技能格。命中、精确格挡、破势和证据确认各有不同的声音/动画语言。

### 4.2 工灯状态

工灯界面同时表达：当前可照见目标、已验证证据、待验证假设、可用工印和归位准备度。

- 未知目标用轮廓/方向暗示，不显示精确答案。
- 正确观察给出材质、声响与简短关键词；不弹长段教程。
- 错误假设保留为可撤销标记，不惩罚到卡死。
- 归位窗口必须同时用世界状态、声音和 HUD 表达，并支持延长窗口辅助。

### 4.3 Boss 与公共工程

Boss 顶栏分离生命/抵抗、失序和阶段目标。公共工程可用节点图代替血条，例如三座闸室的水位和锁定状态。阶段切换前给出可读过渡，不用突然清除玩家进度。

## 5. 探索与交互

交互提示格式固定为“输入 + 动词 + 对象”，例如“E 检查伞骨”，不只显示“互动”。多个对象重叠时按视线、距离、任务相关性和玩家最近选择排序，并允许方向键切换。

导航采用递进提示：自然地标/角色语言 → 指南针扇区 → 地图区域 → 可选精确路径。玩家可分别调整主线、支线和探索提示强度。

拾取提示按重要性分层：普通材料合并显示；新工艺域、任务物和独特物品提供说明；批量拾取不连续打断移动。

## 6. 深层系统

### 6.1 地图

地图按地区 → 子地区 → Cell/室内分层。默认显示已知地标、玩家标记、公共工程和可预测日程，不铺满未知问号。筛选器记忆上次选择，并提供图标+文字双重区分。

### 6.2 天工录

天工录的四栏为：器物/工艺、人物/组织、地区/工程、证据/异议。词条显示“亲见、转述、推断、异议”来源，不把未验证传闻写成事实。玩家可在主文、附注和保留空白之间裁定。

### 6.3 行记与任务

行记按“当前承诺、可继续线索、等待世界变化、已归位”组织，不用几十条同级任务挤在一起。每条任务显示玩家动词、最近证据、相关人物与暂停原因。

### 6.4 背包与工艺

按用途而非稀有度颜色优先分类。材料支持搜索、来源/用途反查、锁定、批量和“缺什么”清单。工艺界面先表达用途与权衡，再展示数值变化；失败结果提前说明可恢复程度。

### 6.5 人物关系

不显示单一爱心进度条。人物页展示当前关系描述、重要共同经历、可用专业协作、未解决分歧和大致日程。底层维度只在高级辅助/调试视图显示。

## 7. 输入基线

| 动作 | 键鼠默认 | 手柄默认 | 规则 |
| --- | --- | --- | --- |
| 地面八向移动 | W/A/S/D；鼠标用于 UI | 左摇杆 | 输入相对受控相机映射到世界地面平面；镜头方位切换时不突变已提交方向 |
| 纵跃/跨越 | Space | A | 可变高度；只跨越明确可达表面；提供离地宽限和提前输入，不承担下穿平台语义 |
| 轻击/重击 | 左键/右键 | X/Y | 可交换，支持输入缓冲 |
| 守势 | Shift | LB | 按住/切换可选 |
| 闪身/风渡 | Left Ctrl 或 C | B | 与跳跃分离；长按与连按不承担不同关键功能 |
| 武器技 | Q | RB | 情境图标说明当前架势能力 |
| 工灯 | E | LT | 按住聚焦；切换模式可选 |
| 交互 | F | RT | 与跳跃、攻击和工灯分离，避免重叠对象误触 |
| 架势切换 | 1/2/3 或滚轮 | 十字键左/右 | 支持顺序切换与直接选择；只作用于当前武器 |
| 切换武器/武器轮盘 | V；长按打开轮盘 | 十字键上；长按打开轮盘 | 获得剑器后显示；Tab 永远留给 DOM 焦点；仅在 04 允许的自由状态提交，不能与架势/物品/交互复用 |
| 快捷物品 | R | 十字键下 | 长按轮盘可选 |
| 地图/行记 | M/J | View 短按/长按 | 短按在释放且未越过 350 ms 时开地图；越过阈值只开行记，不得先开地图再跳页；Menu/Start 保留暂停 |

所有键可重映射，冲突时给出原因和替代建议。组合键不作为完成主线的唯一方式。高频按压提供按住替代；QTE 可关闭或自动完成。

触控在获得第二武器前不占常驻按钮；解锁后由可重定位的情境切换按钮/长按轮盘承载，并计入“右侧不超过 4 个常驻动作”的预算。武器轮盘、快捷物品轮盘和架势提示共享视觉组件但拥有不同 ActionId/焦点上下文，不能因长按阈值误触另一个系统。

### 7.1 触控与移动端门

移动端不是简单显示虚拟摇杆。触控方案至少包含：左侧可重定位移动区、右侧不超过 4 个常驻动作、情境动作合并、手势冲突隔离、横屏安全区、震动开关和外接手柄切换。触控完整通关序章与一个章节 Boss、内存/温度达标后，移动浏览器/原生端才可进入 Beta。

网页必须处理浏览器保留快捷键、输入法、窗口失焦、手柄断连和页面隐藏。Tab 只做 DOM 焦点导航；Escape 若正用于退出全屏/Pointer Lock，该次不再触发暂停或返回。普通文本和 IME 分别走 `beforeinput/input` 与 composition commit，均只进入文本上下文。失焦立即清空持续按键并暂停；恢复时不补算后台经过的真实时间。

## 8. 可访问性基线

### 8.1 视觉

- UI 缩放 80%–200%，正文默认不低于 18 px 的 1080p 等效尺寸。
- DOM 控件使用语义元素、可见焦点、ARIA 状态与合理标题层级；Canvas 内不可访问的信息必须在 DOM 辅助层提供等价摘要。
- 高对比主题、背景遮罩、图标轮廓和多种色觉配置。
- 关键信息不只依赖颜色；敌我、可交互和证据同时用形状/动画/声音区分。
- 镜头抖动、屏幕闪烁、运动模糊、景深、色差和粒子密度可独立调节。
- 字幕支持字号、说话人、背景、重要声音字幕和方位箭头。

### 8.2 听觉

- 主音量、语音、音乐、环境、战斗和 UI 独立控制。
- 关键声纹谜题有视觉/触觉等价提示，但保留声音作为体验增强。
- 单声道、动态范围预设、耳鸣音替代和重要声音字幕。

### 8.3 操作与认知

- 输入重映射、按住/切换、死区、连按替代、自动镜头和瞄准辅助。
- 精确格挡/归位窗口可分级延长；游戏速度可在单机战斗中降至 70%/50%。
- 可选路线提示、任务摘要、近期对话回看、证据自动整理和教程重播。
- 难度选项描述具体改变，不用价值判断标签。

### 8.4 验证要求

每个核心流程用纯键盘、纯手柄、200% UI、无颜色、静音+字幕、低镜头运动至少各完成一次。辅助模式必须覆盖序章、两个武器教学、三地区 Boss 和结局。

## 9. 状态与焦点

- 模态打开后焦点移入，关闭后返回触发控件；键盘/手柄焦点始终可见。
- 隐藏 UI 不得继续接受输入或被屏幕阅读层读到。
- `visibilitychange`、`pagehide`、全屏退出和 WebGL context lost 必须进入明确状态机；页面后台停止渲染并提交本地存档事务。
- 同一账号开多个标签页时通过 Web Locks/BroadcastChannel 或等价平台服务选出一个写入者，其他标签只读或要求接管。
- 异步加载显示可取消状态、当前阶段和失败恢复；不能无限转圈。
- 失败覆盖层默认聚焦“重试”，同时提供更改难度、读取存档、返回安全点和退出。
- 完成覆盖层不强制退出地区；提供继续游览、查看记录和进入下一章。

## 10. 教程

教程遵循“安全试一次 → 正式用一次 → 组合变式一次”。短提示不超过两行，玩家可立即重做。未使用的高级提示不提前轰炸；连续失败后提供针对性提示而非重复同一句话。

教程内容由正式输入、能力和本地化数据生成，不手写键名。重映射后所有提示同步更新。

## 11. 本地化与文本

- 所有玩家文本用本地化 key；复数、称谓、数字与按键使用参数化格式。
- UI 预留简体中文基准宽度的 1.6 倍，关键按钮不依赖固定字符数。
- 文本截断必须可查看完整内容；不以缩到极小字号解决溢出。
- 对话、任务和天工录支持版本标记，语音锁定后修改需重新核对时间轴。

## 12. UI 组件与主题

DOM 组件以状态而非页面复制：Button、Toggle、Tab、List、Tree、Card、Tooltip、Meter、Toast、Modal、EvidenceNode。每个组件统一定义 default/hover/focus/pressed/disabled/loading/error 和键鼠、手柄、触控、屏幕阅读行为。必须贴近角色/场景的 Radial 或世界标记由 Axmol 绘制，同时向 DOM 辅助层发布可操作镜像。

视觉语言取自纸、墨、木、金属与灯，但不使用低对比仿古纹理牺牲可读性。地区可以换装饰和动效，交互语义与布局不随地区变化。

## 13. UX 发布门禁

- 新玩家在无口头指导下完成序章核心动作。
- 95% 的测试者能在 10 秒内找到地图、行记、设置和退出。
- 战斗失败回到可操作状态不超过 12 秒（不含首次冷加载）。
- 核心交互在键鼠和手柄之间功能对等。
- 1080p、1440p、4K、16:10、21:9 与 200% UI 无阻断溢出。
- Chrome/Edge/Firefox 支持矩阵下 DOM 与 Canvas 对齐，刷新、全屏、DPR、浏览器缩放和页面隐藏不丢焦点/输入。
- 无鼠标、无声音、无颜色、仅屏幕阅读菜单路径和低性能档均能进入/保存/退出游戏。
- 字幕、色觉、高对比、减震、输入辅助通过专项清单。
- 深层面板暂停与焦点恢复无输入泄漏。

## 14. 页面与 Session 流程

```text
能力检测
  ├─ 不满足 WebAssembly/WebGL2/IndexedDB → 可解释阻断与帮助
  └─ 满足 → 版本/缓存检查 → 标题页
                              ├─ 继续本地档
                              ├─ 新建 Guest
                              ├─ 导入档
                              └─ 可选登录/云同步
                                      ↓
                            内容依赖与真实加载进度
                                      ↓
                     游戏内探索/战斗/对话/深层面板
                                      ↓
                      安全保存 → 章节结算/返回标题
```

标题页必须始终能进入设置、辅助、存档支持和退出；登录不能盖住“本地继续”。版本更新时先说明下载量、是否需重启和本地/云状态，再让玩家选择安全切换。加载页显示 Shell/WASM/核心包/地区包/存档迁移的真实阶段，失败提供重试、诊断和回到上一安全版本，而不是一个没有含义的百分比。

## 15. DOM、Canvas 与状态所有权矩阵

| 界面 | 主要承载 | 权威状态 | 备注 |
| --- | --- | --- | --- |
| 世界角色、命中、机关、世界标记 | Axmol/Canvas | GameCore 快照 | 世界标记向 DOM 辅助层发布等价描述 |
| 战斗 HUD、字幕、交互短提示 | DOM 覆盖层 + 少量 Canvas 世界反馈 | GameCore UI 摘要 | DOM 只显示/提交 Action，不计算资源 |
| 地图、行记、天工录、背包、关系、工艺 | DOM | GameCore 面板模型 | 打开/关闭的暂停语义由 GameCore 确认 |
| 标题、Profile、设置、账号、同步冲突 | DOM/Web Shell | Platform + GameCore/Sync DTO | 不把 OAuth token 放入游戏面板模型 |
| 失败、归位、章节结算 | DOM + Canvas 演出 | GameCore 结果事务 | 先本地提交，再显示已保存；奖励只结算一次 |
| 调试、实体/性能 HUD | 独立开发层 | 各模块只读诊断 | Release 默认剔除，不能复用玩家按钮 |

Canvas 与 DOM 之间只传稳定 `UiEvent/UiCommand` 与世界坐标投影。DOM 不按每帧完整复制实体树；Canvas 也不栅格化长文本菜单来绕过可访问性。

## 16. 保存与同步用语

玩家不应理解 revision、Operation 或事务才能判断安全性。统一使用：

| 状态 | 玩家文案语义 | 禁止文案 |
| --- | --- | --- |
| 本地事务成功 | “已保存在此设备” | 仅显示模糊云图标 |
| 已登录且云端确认 | “已同步至云端” + 时间/设备 | 把本地成功写成“同步成功” |
| 离线/服务暂时失败 | “已保存在此设备；联网后继续同步” | “保存失败”或阻止离开 Boss 结算 |
| 本地写入失败 | “尚未安全保存”，提供重试/导出/空间帮助 | 继续显示“已保存” |
| 双端叙事冲突 | 展示两设备的进度摘要、影响和保留/复制选择 | “新版/旧版”“覆盖/取消”而无后果说明 |
| 浏览器存储可能被清理 | 提示云同步/导出与持久存储状态 | 宣称 IndexedDB 永久安全 |

危险操作（覆盖导入、删除 Profile、清站点数据、放弃冲突分支）必须先显示影响并提供可行的导出/保留路径。

## 17. F1“雨夜试伞”UX 验收路径
