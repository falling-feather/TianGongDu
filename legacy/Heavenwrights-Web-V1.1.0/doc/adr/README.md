## 记录规则

- 重大技术决策先写 ADR，再落到实现或第三方依赖接入。
- 已接受 ADR 不直接改写历史；需要变更时新增替代 ADR，并在旧 ADR 中标注替代关系。
- 普通任务、每日记录和短期 bug 修复不写 ADR，进入规划、测试或发布历史。
