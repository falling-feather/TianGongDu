## 文档边界

本文件负责回答“下一步做什么、为什么做、怎样算完成”。已完成事实进入 `../CHANGELOG.md`，长期技术规则进入 `01_开发者手册.md`，全局设计准绳进入 `03_项目全局设计规划.md`；世界观和纵切设计的细节以 `07_江南雨巷纵切规格.md`、`10_世界观构建方案.md` 和 `11_项目设计记忆快照.md` 为支撑依据。

## 背景与问题

项目已完成 M0 浏览器原型、内容数据样例、Node 静态测试、Python 内容校验和前期设计文档，并已把 `07_江南雨巷纵切规格.md` 扩展为江南第一版投放设计书。下一阶段的核心问题不是继续扩散设定，而是把“江南雨巷 + 纸伞门径”从文档蓝图推进到可反复验证的纵切基础：内容 Schema 更清晰、交互可验收、工程边界稳定、未来 Axmol/C++ 接入有入口。

## 阶段目标

1. M0 demo 能稳定运行并通过自动测试、内容校验和人工浏览器冒烟。
2. 江南雨巷纵切的内容对象、交互路径和验收口径能被文档追踪。
3. 内容层继续保持文本化、可 diff、可校验，并避免中文显示文本写死在逻辑数据里。
4. 工程目录保持“当前浏览器原型”和“未来 Axmol/C++ Runtime”边界清楚。
5. 重大技术取舍进入 ADR，避免只存在聊天记录或临时 README 中。

## 依赖与风险

- Axmol WebAssembly/WebGL2 链路仍需技术冒烟，不能只依赖当前无外部依赖的浏览器原型判断最终可行性。
- 文化资料需要二次核验；`08_资料索引与调研计划.md` 中待查内容不能直接视为最终设定。
- 内容 Schema 当前已有江南 M1 内容包测试和本地化 key 覆盖，但 Python 校验器仍是结构烟测，尚未覆盖跨文件引用、数值上下限和循环引用。
- demo 当前以 Canvas 和静态资源验证表达，不代表最终性能、资源包、音频解锁、存档或移动端适配已经达标。

## 不做事项

- 本阶段不做完整全国大地图。
- 本阶段不做 MMO 或实时联机。
- 本阶段不引入大型二进制编辑器工作流。
- 本阶段不把当前 browser demo 误认作最终 Axmol 实现。
- 本阶段不直接复制大型第三方依赖进仓库；正式接入前先写 ADR。

## 验收方式

- 自动测试：运行 `npm test`。
- 内容校验：运行 `npm run validate:content`。
- 浏览器冒烟：运行 `npm run dev`，打开 `http://127.0.0.1:4174/`。
- 人工路径：检查加载非空、移动、冲刺、架伞、收风、借风、交互、夜市选择、归位和重置。
- 文档检查：README 能导航到规划、开发者手册、测试指南和发布历史；已完成事实写入 `CHANGELOG.md`。
