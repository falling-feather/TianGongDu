## 1. 版本全貌

1.0 由三部地区篇章和一个短序/尾声连接。每个地区主线目标 315 分钟，另有约 150 分钟成链支线和大量回访/精通内容。三地区的玩法不是同一个模板换皮：

| 地区 | 玩家学习 | 武器/系统重心 | 地区问题 | 主线 |
| --- | --- | --- | --- | ---: |
| 江南水路 | 看见劳动与公共维护 | 伞器、基础兵击、工灯定印 | 无人署名是否等于无人负责 | 315 分钟 |
| 龙泉山炉 | 质疑单一标准与师承荣耀 | 剑器、工艺权衡、动态调校 | 锋利是否高于用途 | 315 分钟 |
| 徽州墨岭 | 使用记录并容纳异议 | 契约、关系、非战斗裁定、跨区组合 | 谁能书写规则与留白 | 315 分钟 |

活动时间目标：探索与空间推理 28–32%，战斗 22–28%，角色与叙事 18–24%，工艺/调查/裁定 18–24%，不可交互过渡不高于 8%。这些比例用于盲测诊断，不要求每分钟机械平均。

## 2. 跨地区主线

序章在天工渡完成护牒、工灯和《天工录》残页教学。江南、龙泉、徽州各提供一组看似相同、实际语义不同的“无款工印”。玩家的记录选择形成三个长期维度：

- **署名原则**：个人、师门、共同体或保留异议。
- **用途原则**：效率、安全、传承或公共可及。
- **介入原则**：天工院标准、地方自治或共同试行。

三个维度不直接显示善恶分数，而影响终章证据、NPC 支持、工艺配方、地区后日常和《天工录》词条。任何完整路线都能结束主线；最丰富结局要求玩家真正理解三地，而非收集隐藏点数。

## 3. 第一部：江南水路

### 3.1 地区定位

江南是低平水网和密集熟人社会。雨、伞、染色、旧桥和潮闸把个人手艺与公共维护连在一起。玩家从纸伞小故障进入，最终面对外圩倒灌与“责任被切成太多小块”的地区问题。

系统递进：移动/交互 → 伞器檐守 → 精确格挡 → 基础照见 → 引序/定印 → 关系协作 → 大型公共工程归位。

## 4. 第二部：龙泉山炉

### 4.1 地区定位

龙泉沿山脊、矿谷和地火组织生产。剑器被视为荣耀中心，但真正维持它的是选矿、运料、炉工、磨工和安全规则。第二地区必须证明框架可复用：不用重写角色、任务、战斗和存档系统，就能增加全新武器、垂直关卡和锻造语法。

系统递进：剑器试锋 → 材料取舍 → 藏锋反击 → 矿区垂直探索 → 回炉动态调校 → 移动炉车 → 多场地地区战。

## 5. 第三部：徽州墨岭

### 5.1 地区定位

徽州以货单、林契、谱库、砚田和远行商路把分散社区连接起来。这里减少连续战斗，增加可失败的交涉、证据板、同伴调度和非战斗裁定，但保留动作系统压力。玩家必须理解：记录能保护弱者，也能把没机会书写的人排除在外。

系统递进：货单核验 → 建筑受力与口述证据 → 水磨/田渠共同体 → 动态契约改变场地规则 → 公开辩契 → 商队调度 → 三地区记录总裁定。

## 8. NPC 交互框架

每次核心 NPC 交互从通用系统组合，不写角色专属 UI：

- **相遇层**：日程、天气、章节、地点和当前工作决定可交互状态。
- **关系层**：信任、认同、亏欠、熟悉和声誉决定语气、请求和协作。
- **专业层**：工艺、研究、交易、指导、测绘、谈判等角色能力。
- **同行层**：短时场景协作或有限伙伴，不做全员永久战斗队伍。
- **后果层**：任务、地区归位和《天工录》记录改变行为与空间。

玩家不能靠重复送礼刷满所有关系。高阶关系需要共同经历、价值判断和承担责任。

## 10. 奖励与跨区复用

- 江南奖励“如何保护和导流”：伞器、天气、公共工程协作。
- 龙泉奖励“如何权衡和调校”：剑器、材料、动态构筑。
- 徽州奖励“如何记录和组织”：契约、同伴调度、证据组合。

跨区组合示例：伞器风渡为龙泉移动炉车导风；剑器回炉能在徽州墨契规则下临时改变材料标签；莫鸣声纹帮助书院识别被改写档案；韩问泉地图与小满信路共同开启白岳安全路线。组合提供新解法，不强迫玩家回到每个旧角落清图。

## 11. 结局结构

地区结局由“设施归位 + 责任安排 + 署名方式 + 异议保留”四项组成。1.0 尾声组合三地结果，形成 6–9 个主要尾声变体和角色来信差异，不追求指数级全排列。

所有合格结局都保留世界的平和底色和继续更新空间。最差结果是保守、昂贵或关系疏远，不用突然毁灭地区否定几十小时投入；最完整结果也不宣称所有矛盾永久消失。

## 12. 制作切分

每个地区依次经历：枢纽灰盒 → 探索语法验证 → 核心 NPC 网络 → 一条主线纵切 → 全子地区 → 支线与回访 → 归位后状态 → 美术/音频/优化。不得先铺满地图再补玩法，也不得先写完全部台词再验证任务状态机。

M1 江南是质量金标准。M2/M3 允许在同一模板上更快生产，但不能把节省的工程时间变成内容缩水；省下的时间应投入独特机制、角色关系和回访状态。
