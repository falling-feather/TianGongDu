# 12｜内容、存档与版本契约

> 上级文档：[`../01-开发者文档.md`](../01-开发者文档.md)
> 状态：Accepted Baseline
> 最后更新：2026-07-10

## 1. 四种数据形态

1. **Authoring Source**：便于人和编辑器修改的 JSON、对话图、场景标记和 DCC 源文件。
2. **Validated Model**：通过 Schema、跨引用、语义和预算校验的内存模型。
3. **Baked Package**：规范化、索引化、带哈希和版本的发布数据/PCK。
4. **Runtime State**：从不可变定义实例化的 Session 状态与存档差异。

游戏不能在 Release 中随意读取 Authoring Source。编辑器预览与最终构建必须共用同一验证/烘焙代码，避免“编辑器能跑、发布包坏掉”。

## 2. Schema 规则

- 使用 JSON Schema Draft 2020-12。
- 每类根对象有 `$id`、`schemaVersion`、`additionalProperties` 策略和最小示例。
- 公共结构放 `$defs`，但避免一层字段改动让所有 Schema 继承爆炸。
- 单位进入字段名或结构，例如 `durationTicks`、`distanceMeters`。
- 枚举用于稳定机器语义；玩家文本用本地化 key。
- 默认值只在 Schema/烘焙器一处定义，编辑器展示但不另存第二套默认。
- Schema 通过只是第一层；跨引用、图可达、预算、语义和包依赖由校验器补充。

## 3. 包清单

每个内容包至少包含：

```json
{
  "packageId": "region_jiangnan",
  "packageVersion": "1.2.0",
  "schemaSetVersion": "1.1.0",
  "contentApiVersion": "1.0",
  "executableRange": ">=1.0.0 <2.0.0",
  "dependencies": [
    { "packageId": "core", "range": "^1.3.0" }
  ],
  "entryIndexes": ["indexes/entities.bin", "indexes/quests.bin"],
  "files": [
    { "path": "data/jiangnan.bin", "sha256": "...", "size": 123 }
  ],
  "signature": { "algorithm": "ed25519", "keyId": "release-2026-a", "value": "..." }
}
```

确切签名算法在安全烘焙后锁定；示例表达合同，不应在未评审情况下直接复制为实现。

加载器先建立完整依赖图、检查循环/范围/哈希/签名，再挂载任何包。加载顺序由依赖拓扑和稳定排序决定，不允许文件系统顺序影响覆盖。

## 4. 版本线

项目维护五条独立版本：

| 版本 | 说明 | 变化示例 |
| --- | --- | --- |
| ExecutableVersion | 客户端代码和平台包 | 战斗系统实现、Godot 升级 |
| ContentApiVersion | Runtime 可理解的内容语义 | 能力节点或任务节点新增 |
| SchemaSetVersion | Authoring 文件结构 | 字段拆分、枚举变化 |
| PackageVersion | 单个内容包 | 江南修复或本地化更新 |
| SaveVersion | 存档结构 | 世界状态、任务状态迁移 |

版本不能共用一个数字假装同步。兼容范围明确写在包和存档头中。

## 5. 稳定 ID 注册

构建生成全局 ID 索引：ID、类型、拥有包、引入版本、弃用版本、替代 ID、显示名 key、来源文件。跨包引用必须指向目标包公开导出表。

ID 变化流程：

1. 尽量保持原 ID，只改显示名或路径。
2. 必须替换时，旧 ID 进入 alias/tombstone 表。
3. 迁移器在内容和存档层都转换，并记录一次性诊断。
4. 至少跨一个大版本保留别名，删除需审查现存存档样本。

## 6. 存档结构

存档由小型 Header、压缩 Payload 和校验/备份组成：

```text
SaveHeader
├── magic / format / saveVersion
├── executableVersion / contentApiVersion
├── packageSet (id, version, hash)
├── slot / profile / createdAt / playedTicks
├── region / checkpoint / difficulty / accessibility profile id
└── payload hash / compression / migration history

SavePayload
├── global state
├── region states
├── persistent instances
├── player progression and inventory
├── quests / relationships / records
├── deterministic random streams
└── optional diagnostics summary
```

设置与存档分离：显示、音量、输入映射和大部分辅助选项属于 Profile；难度中会影响规则的部分在存档记录。

## 7. 保存事务

1. 在模拟安全点冻结需要持久化的不可变快照。
2. 后台序列化到临时文件。
3. 校验可读性、哈希和必要不变量。
4. 将现有主档轮换为备份。
5. 原子替换主档并刷新目录项。

游戏保留最近 3–5 个自动保存轮换、章节/地区关键点和玩家手动槽。禁止先覆盖唯一档再验证。存档 UI 显示版本、地区、时间和内容包异常。

## 8. 迁移

迁移是从 `SaveVersion N` 到 `N+1` 的纯、可重复转换。跨多个版本按链执行；每一步有黄金样本和反向不可用说明。迁移不得访问网络或当前场景树。

流程：复制原档 → 在隔离目录迁移 → 完整反序列化/不变量验证 → 写新档 → 保留原档。失败进入只读诊断，不覆盖。

内容包回滚时，如果新存档已使用不可逆内容状态，加载器应选择兼容旧包、隔离相关状态或明确拒绝；不能静默删除玩家进度。

## 9. 内容补丁与扩展

基础内容定义默认不可覆盖。官方修复通过声明式 Patch：目标 ID、允许修改的字段、前置版本、操作、结果哈希。高风险字段（类型、所有权、持久身份、脚本节点）不能热补丁，必须提高 Content API/可执行版本。

模组使用 `mod.<author>.<name>.*` 命名空间。1.0 仅能新增或在公开扩展点附加数据/资源，不能覆盖核心 ID、加载任意程序集或修改存档头。

## 10. 本地化契约

本地化 key 稳定且带领域，例如 `npc.shen_yan.name`, `quest.jn.umbrella_four_seasons.title`。源文本不是 key。格式参数有名字和类型，构建检查所有语言的参数一致。

对话/字幕附带说话人、时间预算、语音资产 ID 和版本。删除文本保留 key 墓碑直到所有引用和旧存档安全迁移。

## 11. 内容错误分级

| 级别 | 示例 | 行为 |
| --- | --- | --- |
| Fatal | 核心包签名/哈希错误、依赖循环、未知 Content API | 阻止启动，提供修复/安全模式 |
| Package Fatal | 可选地区包损坏 | 隔离包；不允许载入依赖它的存档 |
| Content Error | 未知 ID、任务图不可达、Boss 无出口 | 构建失败；Release 不应出现 |
| Warning | 未使用定义、预算接近阈值 | 构建报告；负责人确认或修复 |
| Info | 自动默认、规范化差异 | 可审查报告 |

错误包含文件、行列/JSON Pointer、对象 ID、包、规则编号和修复建议。批量错误去重但不只显示第一条。

## 12. 验证层

1. JSON 语法与 Schema。
2. 全局 ID 唯一、类型和跨包可见性。
3. 引用存在、依赖闭包、循环和版本范围。
4. 任务/阶段/对话图可达性与终点。
5. 数值单位、上下限、预算和性能档。
6. 本地化覆盖与格式参数。
7. 资产存在、导入设置、哈希与许可。
8. 运行时沙盒加载与存档往返。
9. 内容规模和地区时长门禁。

当前 Foundation 校验器先覆盖目录层，F1 扩展为完整构建链。

## 13. 确定性与规范化

烘焙器对对象键、ID 索引和输入文件稳定排序；输出不包含当前时间、绝对路径或机器相关值。相同 Git 提交和工具版本应产生相同清单/数据哈希（平台特定压缩产物另行记录）。

CI 对烘焙结果做二次构建对比，发现非确定性立即失败。生成文件禁止手工修改，Header 记录生成器版本和源摘要。

## 14. 恢复与兼容测试

- 每个支持的 SaveVersion 有黄金档。
- 每个正式包版本保留最小兼容样本。
- 测试缺包、包降级、哈希损坏、磁盘满、写入中断和权限拒绝。
- 任务、关系、Boss 中途、Cell 边界和重大选择各有存档往返。
- 100 次连续保存/加载不增长不可回收状态。
- 随机抽样正式玩家匿名诊断档前必须获得明确同意并移除个人信息。
