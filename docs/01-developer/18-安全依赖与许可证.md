# 18｜安全、依赖与许可证

> 上级文档：[`../01-开发者文档.md`](../01-开发者文档.md)
> 状态：Active Baseline
> 最后更新：2026-07-10

## 1. 保护目标

- 玩家设备、文件与隐私。
- 存档完整性和可恢复性。
- 官方可执行、内容包、更新清单和发布身份。
- 源代码、源资产、签名密钥和第三方授权。
- 构建/发布供应链与远端仓库。

单机游戏不需要把玩家当对手。防作弊不是 1.0 安全目标，不能用内核驱动、侵入式监控或破坏模组/离线体验换取不存在的强联网经济完整性。

## 2. 信任边界

不可信输入包括：下载清单/包、模组、存档、配置、命令行、剪贴板/文本导入、DCC 资产、对话表格、第三方库和 CI 产物。官方 Git 中的内容也必须验证，不能因“来自自己”跳过解析边界。

Godot PCK 是容器，不是安全边界；JSON Schema 是结构验证，不是全部语义/资源限制。

## 3. 内容解析

- 解析前限制文件/解压后大小、数量、深度、字符串长度、数组长度和图节点数。
- 路径规范化后必须留在允许根；拒绝绝对路径、设备路径和 `..` 逃逸。
- ZIP/PCK 防目录穿越、重复路径、大小欺骗和解压炸弹。
- ID、枚举、数值单位和引用严格验证；未知高风险字段失败。
- 图遍历有节点/步数上限；正则和表达式使用受控实现。
- Release 不反序列化任意类型、不执行内容中的脚本/命令。

## 4. 更新与签名

清单和包使用独立数字签名与 SHA-256 或经评审更强哈希。签名私钥不在仓库、客户端或普通 CI 变量中；使用受控密钥服务/硬件或最小权限签名作业。

密钥有 ID、用途、启用/过期、轮换和撤销。客户端内置根信任并能接收受签名的轮换链。构建产物在签名前完成测试，签名后再做安装冒烟并验证签名仍有效。

TLS 保护传输，但不能替代产物签名。下载域名和重定向受限。

## 5. 存档安全

存档不是秘密，可由玩家控制；加载时按不可信输入处理。限制集合与字符串、验证版本/包/ID、拒绝非法数值和循环引用。损坏档不崩溃、不覆盖备份。

不为了“防改档”加伪加密或远端验证。完整性哈希用于检测损坏，不宣称阻止玩家修改。若未来有云同步，冲突处理优先保留数据。

## 6. 模组隔离

1.0 不加载任意代码、原生库、Shader 源或网络脚本。资源类型、尺寸、Shader/材质能力、音频时长和场景节点白名单受控。模组挂载到命名空间，不能覆盖官方路径。

启用前显示来源、版本、权限（1.0 应几乎没有系统权限）与存档影响。安全模式禁用模组并保留原档。官方不承诺恶意本地管理员无法修改游戏，而承诺游戏自身不主动给模组任意执行能力。

## 7. 秘密管理

- 仓库和发布包不得包含私钥、服务令牌、个人访问令牌和生产连接串。
- 本地开发使用忽略文件/系统凭据库；提供无秘密示例配置。
- CI 使用环境级最小权限、短期令牌和受保护分支/标签。
- 日志屏蔽 Authorization、Cookie、签名材料和隐私字段。
- 秘密扫描在提交与 CI；发现泄露先轮换，再删除历史中的暴露。

客户端内的任何 key 都视为公开，不能用来保护服务端权限。

## 8. 依赖准入

新增依赖必须回答：为何标准库/现有依赖不够、维护活跃度、许可证、下载来源、传递依赖、平台支持、AOT/裁剪、包大小、已知漏洞、替代/移除成本。

锁定精确版本和哈希；依赖源白名单；PR 显示 lockfile 与 SBOM 差异。自动更新机器人可以提 PR，但不能绕过测试自动合并大版本。

高风险依赖（解析器、网络、加密、压缩、原生插件、编辑器扩展）需要安全审查和恶意输入测试。

## 9. 漏洞管理

CI 扫描 NuGet、npm、Python、Godot/原生库和容器。结果按可达性、利用条件和玩家影响分级，不只看 CVSS 数字。Critical/High 可达问题阻断发布；不可达例外有证据、负责人和到期日。

安全更新优先补丁分支、回归、签名发布和明确说明。Godot/.NET 等基础平台在支持期内跟进安全补丁，但不在内容锁定期盲升大版本。

建立私密漏洞报告渠道后写入 SECURITY.md；公开仓库 Issue 不要求报告者披露可利用细节。

## 10. 许可证与素材来源

每个第三方代码、字体、音频、纹理、模型和参考资产记录：名称、版本/来源、作者、许可、修改、归属要求、分发范围和证明文件。不能用“网上找的”“AI 生成所以无版权问题”作为来源。

发布自动生成第三方通知，检查 copyleft/署名/商用/再分发兼容性。强 copyleft 或限制性素材进入前由负责人评估，不在快发布时才发现无法分发。

AI 生成资产记录工具/模型、日期、提示摘要、输入参考权利、人工修改和审核。不得用未授权在世艺术家/品牌资产作为直接复制目标；涉及真实工艺和文化资料遵循 05/09 的研究规则。

## 11. SBOM 与产物证明

每个候选版生成机器可读 SBOM，覆盖托管、原生、引擎、工具运行时和随包分发资源组件。Release Record 关联 commit、工作流、runner 镜像、依赖锁、SBOM、产物哈希、签名和测试。

构建与签名权限分离；普通 PR 不能访问发布密钥或 Stable 渠道。来自 fork/外部贡献的工作流不执行秘密上下文。

## 12. 隐私

默认不收集非必要数据。任何遥测先写目的、字段、法律基础/同意、保留、访问、删除和第三方处理。设置中可查看并关闭；关闭不影响单机内容。

崩溃包在本地脱敏，避免用户名、绝对路径、输入文本、完整存档和模组内容。玩家主动附加文件时明确预览和说明。

未成年人、不同地区法规和商店平台要求在实际发行市场确定后由专业人员审阅；本文档不是法律意见。

## 13. 威胁场景验收

- 恶意包路径逃逸、解压炸弹、巨大图和未知资源类型被拒绝。
- 清单被篡改、签名错误、旧版本重放和下载中断安全失败。
- 恶意/损坏存档不覆盖备份、不执行代码、不无限分配。
- 模组无法读取任意文件、联网、加载 DLL 或覆盖官方 ID。
- 日志/崩溃包不含秘密和常见个人路径。
- CI fork 无发布秘密，发布只来自受保护 tag/工作流。
- 依赖/许可证/SBOM 差异在 PR 和候选版可审查。

这些场景在 16 的夜间/候选门中自动或人工执行。
