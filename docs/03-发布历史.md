## 2.1.0-foundation.1｜C++ Web 与多端同步技术修订｜2026-07-10

### 已完成

- 根据项目负责人新决策，将正式平台改为浏览器首发的横版动作 RPG，不再采用 Godot/.NET 桌面优先方案。
- 选定 C++20 游戏核心、Axmol 2.11.x LTS、Emscripten/wasm32 与 WebGL2；单线程 Web 构建为兼容基线，Pthreads 为条件增强。
- 明确 Canvas 游戏画面与 DOM HUD/菜单分层，加入 PWA、Service Worker、CDN 分包和 IndexedDB 本地存档。
- 设计同一 C++ Gameplay 面向 Web、Windows、Android、iOS 的 Platform/Presentation 边界，避免多端重写玩法。
- 建立可选云同步：访客可玩、本地优先、修订快照、幂等操作日志、领域合并和叙事冲突人工选择。
- 采用 C++20/Drogon + PostgreSQL 作为同步服务参考实现，并用外部 OIDC 身份提供方避免自建密码系统。
- 新增机器可读技术基线与 Schema，并扩展测试守卫 C++、Axmol、Emscripten、WebGL2、DOM、IndexedDB 和同步决策。

### 兼容性

2.1 取代 2.0 的 Godot 技术决策，但不改变世界观、1.0 地区/Boss/NPC 目录和内容生产深度。2.0 ADR 作为历史保留并由 0005–0008 正式取代。

## 2.0.0-foundation.1｜端游重构设计基础｜2026-07-10

### 已完成

- 将正式产品重新定义为 Windows 原生单机动作角色扮演游戏，停止 Web/WASM 优先路线。
- 选定 Godot 4.7 stable .NET + C# 为客户端与编辑器宿主基线，采用分层核心与混合组件模型。
- 建立 00–09 共 10 份顶层文档，以及隶属于 01 的 10–19 二级开发手册。
- 建立 1.0 机器可读目录、两套 JSON Schema 与 8 类模板注册表，固定 3 地区、18 子地区、14 Boss、24 核心 NPC、2 套战斗系统、2 条武器体系和 945 分钟主线预算。
- 建立无第三方依赖的设计校验与 Node 测试，验证数量、稳定 ID、跨区引用、NPC 交互深度、Boss 阶段和文档层级。
- 建立 Runtime / Gameplay / Presentation / Editor 边界、内容包/存档/热更新契约、模板生产和发布门禁。
- 记录四项主线 ADR：桌面引擎、混合组件架构、内容与存档契约、离线优先更新与模组边界。

### 兼容性

这是新主线的设计与工程基础节点，不含正式 Godot 可执行游戏。旧 Web 代码不兼容新运行时，设计意图将按里程碑迁移。
