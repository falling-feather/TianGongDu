## 自动测试

运行：

```bash
npm test
```

当前覆盖：

- `prototype/web-demo/index.html` 包含 demo 标识、核心 UI 文案和操作入口。
- `prototype/web-demo/src/main-v2.js` 包含核心交互函数、冷却、键盘输入、工艺白盒、资源事件和动画循环。
- `prototype/web-demo/src/styles.css` 保持全视口 playfield 和移动端响应式规则。
- `content/**/*.json` 至少包含内容协作必填字段。
- `tests/content-m1.test.mjs` 覆盖江南 M1 内容包的 RegionDef 索引、11 个子地区、7 个核心节点、15 名关键 NPC、10 个主线步骤、8 条支线、10 个建筑、8 类交互、9 类敌人、6 类机关、1 个 Boss、4 个 Boss 阶段、3 个归位结果、美术/音频生产接口和本地化 key。
- `tests/content-map13.test.mjs` 覆盖 `13` 章节化大地图的 6 个大区、36 个点位、`07` 11 个子地区迁移表、自给循环和 7 类编辑器模板。
- `tests/web-demo.test.mjs` 覆盖 Web demo 的 M1 生产面板容器、13 大地图/编辑器入口、资源事件函数、Boss 资源状态函数、染纸晒场工艺白盒、空间交互热点和生产面板样式类。

## 内容校验

运行：

```bash
npm run validate:content
```

当前最低要求：

- 每个 JSON 文件能被解析。
- 每个内容对象包含 `schemaVersion`、`id`、`displayNameKey`、`tags`、`designerNote`。
- `schemaVersion` 为整数。
- `tags` 为数组。

后续应补充：

- 跨文件 ID 引用检查。
- 本地化 key 缺失检查已在 Node 测试中覆盖，后续需要并入 Python 内容校验器。
- 数值上下限检查。
- 循环引用检查。
- 关卡可达性检查。
- 文档和内容 JSON 的中文字段可读性检查，防止误存为乱码编码。
- 资源契约检查：`requiredStates`、`referenceIds` 命名空间、`variants[].sourceQuestId`、资源占位白名单和 `image_example/` 不被运行时代码直接引用。

## 浏览器人工冒烟

启动：

```bash
npm run dev
```

打开：

```text
http://127.0.0.1:4174/
```

最低检查路径：

1. 页面加载成功，Canvas 画面非空，右侧 HUD 和底部技能栏可见。
2. `WASD` 或方向键能移动角色。
3. `Shift` 或 `Space` 能冲刺并消耗气力。
4. `Q` 能架伞，风息能随架伞积累。
5. `E` 能触发收风，影响附近敌人。
6. `R` 能借风越行，靠近风口时能改变位置和记录。
7. `F` 能执行上下文交互，包括检视雨线、请教师傅、检校风口和归位。
8. “接急单”和“完整工序”能改变工牒四目数值与事件记录。
9. “重置试炼”能恢复初始状态。
10. “江南M1”面板能展示美术资源组、音频资源、Boss 阶段 requiredState、白盒接入状态和当前资源事件 ID。
11. “大地图”面板能展示 6 个大区、每区 6 个点位、`07` 迁移说明和自给循环；点击大区卡片后详情能切换。
12. “编辑器”面板能展示 7 类编辑器模板、必填字段、校验规则和 JSON 草稿；导出/重置按钮只影响内存预览，不写回磁盘。
13. 使用架伞、纸刃、伞旋、收风、借风、风铃铆、Boss 显形和 Boss 归位时，资源事件 ID 会随操作更新。
14. 推进 M1 至“竹林选材”后，地图能显示并进入“染纸晒场”；玩家靠近晒场工艺热点时，`F` 上下文交互能切到背包工艺面板并触发 `interaction.jiangnan_rain_alley.craft_paper_oil` 资源事件。
15. 在工艺面板选择“蓝染稳色”或其他可用配方后，材料数量减少，`quest_step.jiangnan_rain_alley.paper_and_oil` 完成，`wind_bell_lane` 解锁，工艺四项指标、NPC 评价、四维数值、伞面颜色和当前资源事件同步变化。
16. 桌面宽屏和窄屏视口下，HUD、技能栏、操作提示、工艺面板、大地图、编辑器和资源生产面板不互相遮挡到不可读。

## 纵切验收口径

`07_江南雨巷纵切规格.md` 中的完整纵切验收仍是后续目标。当前 M0 只要求验证首屏表达和基础交互。进入下一阶段时，验收应至少覆盖：

- 从新游戏到短闭环结束可连续游玩。
- 不依赖开发者口头解释也能理解当前目标。
- 伞师门径与普通近战有明显差异。
- 工艺选择至少影响一个战斗参数和一个视觉或叙事反馈。
- 四维选择至少影响三个后续反馈点。
- Boss 或精英战能验证格挡、弹反、收风、借风越行中的核心组合。
