# 《天工渡》一次性视觉与玩法样例

> 状态：Disposable Gameplay Sample（一次性玩法演示）
> 边界：本目录与正式 Axmol/C++ F1 运行时完全隔离；不作为玩法完成度、存档兼容性或一小时纵切的替代证据。

## 1. 这个“Demo 的 Demo”要回答什么

它不是缩小版正式游戏，而是一场约 30–90 秒、只用于观看与操作的精英战。代码和资产都不承诺复用，只用来回答四个问题：

1. 《天工渡》的镜头是否一眼就是 **2.5D 斜向全景动作 RPG**，而不是俯视角小人、横版平台或自由 3D 漫游？
2. 人物、建筑与场地是否具备接近《斗战神》所强调的尺度、纵深和压迫感？
3. 战斗是否表现为蓄力、位移、受击停顿、击退与阵势变化，而不是轻飘飘的弹幕清怪？
4. HUD 是否只服务战斗与任务，不遮挡主战场，也不呈现网页卡片或手游大厅感？

## 2. 第一指导与原创边界

- 第一指导：以《斗战神》的 **镜头语言、人物尺度、受控全景、空间纵深和厚重战斗阅读性** 为第一参照。
- 补充参照：吸收《暖雪》易读的斜向移动与战斗节奏，但不采用俯视小比例角色或高密度弹幕作为主表达。
- 原创边界：不复刻现有游戏的角色、怪物、场景、图标、字体、剧情或 UI；所有概念均围绕《天工渡》的渡口、伞巷、机巧术与四时祟原创设计。

## 3. 目标画面

### 3.1 镜头与尺度

- 16:9 主画面，基准 1280×720；镜头约 30–38° 斜俯视，但地平线、远山和大型建筑必须可见。
- 玩家在 720p 画面中的站立高度约 150–190 px；普通敌人为玩家的 0.8–1.3 倍，精英为 1.5–2.2 倍。
- 镜头以房间/战斗走廊为单位受控推进，只允许轻微跟随、战斗锁域和关键演出拉远，不提供无约束旋转。
- 前景檐角、伞棚、锁链或雨帘形成遮挡；中景是可行走与战斗平面；远景保留河道、山体、灯火与巨大机械轮廓。

### 3.2 环境

- 首个样例场景：暴雨后的双浦渡—伞巷工位入口。
- 材质关键词：湿青石、油纸伞、旧铜、深色木构、绷紧缆绳、积水倒影、雾雨与暖灯。
- 场景既要有江南水路的真实生活痕迹，也要有机巧术造成的超常尺度；不能只是一条干净漂亮的古镇街。

### 3.3 战斗

- 一名玩家对 3–5 名敌人；一次只突出一个明确威胁，不靠全屏粒子制造强度。
- 基础演示：重击破势、花转轻击衔接、闪避穿位、精英蓄力红线、击退撞入场景物件。
- 每次有效命中至少由姿态变化、受击定格、位移/击退、局部碎屑或水花、声音峰值中的三项共同表达。
- 大招和 Boss 阶段可拉远镜头，但角色仍必须清晰可辨。

### 3.4 HUD

- 左上：玩家生命、架势/资源；右上：一行当前目标与距离提示；下方偏中：4 个核心动作与当前架势。
- Boss 战仅在顶部增加一条长血条；任务对话沿画面下缘短暂出现，不打开大面积面板。
- 视觉语言：深墨、旧铜、米纸白、克制的朱砂警示；形状偏削切与榫卯结构，避免圆角卡片阵列。

## 4. 可运行样例

本目录已实现一个不依赖正式运行时的浏览器样例：

1. 固定斜向镜头、三层景深和随纵深缩放的角色尺度。
2. 一场约 30–90 秒的精英战，支持斜向移动、轻击、重击、闪避和架势切换。
3. 精英会靠近玩家、显示朱砂蓄力圈并执行重击；闪避具有短暂无敌。
4. 玩家与敌人的生命、架势、伤害、击退、胜负和重开均由程序实际计算。

### 4.1 启动

```powershell
cd prototypes/f1-visual-sample
npm install
npm run dev
```

打开终端给出的 `http://127.0.0.1:<port>/`。生产构建使用：

```powershell
npm run build
npm run preview
```

### 4.2 操作

- `W/A/S/D`：在斜向战斗平面移动。
- `J`：轻击；花转势下伤害更高、恢复更快。
- `K`：重击；檐卫势下伤害更高，消耗架势。
- `Space`：闪避并获得短暂无敌。
- `Q`：切换檐卫势 / 花转势。
- `R`：重开本局。

### 4.3 有意简化

- 角色只有一张透明立绘，攻击与受击通过位移、旋转、缩放、染色和镜头震动表达，没有序列帧。
- 场景为预渲染背景，不存在遮挡网格、动态灯光、导航网格或真实 3D 摄像机。
- 雨、水花、攻击弧和敌人警示圈由程序绘制；这是一次性玩法证明，不代表正式美术质量。
- 没有任务、物品、存档、回放、编辑器或一小时内容；这些仍属于正式 F1 工程。
