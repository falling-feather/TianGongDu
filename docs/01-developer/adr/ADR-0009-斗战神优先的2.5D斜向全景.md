# ADR-0009｜《斗战神》优先的 2.5D 斜向全景

> 状态：Accepted
> 日期：2026-07-11
> 决策者：项目负责人、游戏总监、技术/美术基线
> Clarifies：ADR-0005 的宿主用途与表现目标；不取代其 C++/Web/Axmol 决策
> 关联：00 §1.0、02 §3–4/6、04 §3、08 §7/17、10 §17、14 §1/16–17

## 背景与问题

当前总纲、玩法和技术手册仍把正式游戏写成“纯侧视横版、无纵深走位”，并由此派生下穿平台、单战斗线和 Sprite-only 倾向。项目负责人重新确认：目标应是类似《暖雪》所能说明的 2.5D 俯斜可读形式，但整体必须以《斗战神》为第一指导。若不立即统一，关卡坐标、碰撞、输入、相机、角色比例和资产管线会沿错误方向固化。

ADR-0005 选择了 C++/Web/Axmol 宿主，但其当时以横版 2D 为适配理由。本 ADR 只修订产品空间与 Presentation 目标；宿主是否能承担该目标仍须 F1 证据验证。

## 决策驱动因素

- 第一眼必须呈现有纵深、尺度和东方材质重量的全景场面，而非平面卷轴。
- 战斗需要八向走位、绕背、包夹、地面危险和群战层次，同时维持浏览器上的清晰读招。
- 玩家不应持续操作镜头；作者需要可控构图、Boss 竞技场框和跨 Cell 场面调度。
- Gameplay 必须保持渲染器独立和跨 Web/Native 可回放。
- 资产方案必须在画质、下载、显存、动画产能、许可和移动端之间留出实证选择。
- 参考作品只能提供高层设计语言，不能成为复刻、换皮或未授权资产来源。

## 备选方案

### A. 保留纯侧视横版

实现与 2D Sprite 管线最直接，但无法满足已确认的斜向全景、纵深空间与《斗战神》优先方向。拒绝。

### B. 受控镜头的 2.5D 斜向全景

地面平面支持八向/纵深走位，独立高度支持纵跃、击飞、落差和桥层；Presentation 负责斜向投影、遮挡与景深排序。选择。

### C. 自由旋转的完整第三人称 3D

空间自由度高，但会增加镜头操作、场景背面制作、导航/遮挡、网页包体和性能成本，并削弱作者构图控制。拒绝作为 1.0 基线；局部演出镜头不受此限制。

### D. 直接重现参考作品的视觉或关卡

看似能快速对齐，但会破坏《天工渡》自身世界语言并带来版权、商标、许可与产品身份风险。禁止。

## 决策

1. 《斗战神》是整体第一指导，依次约束斜向全景构图、角色/场景尺度、空间层次、东方材质重量、群战组织和受控镜头调度。
2. 《暖雪》是第二参考，只校准俯斜视角下的敌我轮廓、读招和输入简洁度；下级文档不得把它提升为整体风格来源。
3. 正式玩法使用固定方位或作者控制有限方位的俯斜镜头。玩家在地面平面八向移动，不需要持续旋转镜头；Boss、室内和公共工程可用受控竞技场框与平滑切镜。
4. GameCore 权威状态为地面 `x/y`、独立高度 `h` 与离散 `floorLayer`。屏幕坐标、景深排序、遮挡透明度和相机抖动只属于 Presentation。
5. 最终资产可组合 Sprite 帧、骨骼、预渲染层、低复杂度实时 3D 与 Shader。G2 前由同一“雨夜试伞”代表构图实测后锁定；G1 启动宿主关闭未使用 3D 模块不构成纯 2D 决策。
6. 只允许借鉴高层设计语言。角色、场景、地图、UI、剧情、动作、声音、贴图、模型与其他可识别表达必须原创或具备明确许可。

## 正面后果

- 产品、关卡、战斗、美术、UI 和技术共享同一视角语言。
- 地面二维权威加独立高度仍可固定 Tick 回放，避免让渲染树成为玩法真相。
- 受控镜头兼顾全景场面、作者构图和无需额外镜头操作。
- 表现载体延后到代表样片决定，避免在没有浏览器证据时锁死 Sprite-only 或重型全 3D。

## 负面后果与风险

- 纵深排序、脚点、前景遮挡、桥层、目标选择和相机切换比纯横版复杂。
- Sprite 与 3D 混合可能出现比例、光照、透视和动画接触点不一致。
- Axmol 的 WebAssembly 路径仍为 Preview；斜向样片可能暴露宿主扩展、性能或工具不足。
- “参考《斗战神》”若没有原创边界和审阅，容易被误执行为临摹或近似复刻。
- 既有横版文档、夹具和未来实现需要迁移，短期会增加返工。

## 验证与放行门

- G1 的几何启动画面只验构建、生命周期和 WebGL2；不得作为视角/美术通过证据。
- G1 控制样片至少验证相机相对八向移动、纵深碰撞、景深排序、前景遮挡淡化、脚点、目标选择和 context restore。
- G2 前在同一“雨夜试伞”Cell 交付 2–3 套表现样片；画面必须同时包含前景遮挡、中景纵深战斗、远景地标、玩家、两类敌人与证据点。
- 人工盲测能稳定判断可走面、纵深关系、危险范围、敌我轮廓和当前目标；键鼠/手柄在镜头过渡后方向不突变。
- Web/Native 10,000 Tick 回放对 `x/y/h/floorLayer` 一致；Presentation 排序与遮挡不改变碰撞、命中或存档。
- 资产样片附来源/许可、下载、WASM/JS/GPU 内存、draw call、动画产能和多端缩放证据。

## 迁移与回滚

立即同步 00、02、04、08、09、10、11、14、19、技术基线与设计校验；旧发布历史和旧 ADR 正文保留当时事实。F1 任务删除“无纵深/下穿平台”硬约束，改验地面八向、纵深、遮挡与高度/层。G2 表现样片放行前不批量生产最终角色和场景。

若 Axmol 无法满足兼容、性能或工具门，新 ADR 可替换 Presentation 宿主，但继续保持本 ADR 的产品视角和 GameCore 空间合同。只有项目负责人明确改变产品方向并以新 ADR 取代本决策，才可回到纯侧视或自由相机路线。
