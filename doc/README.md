## 阅读摘要

《天工渡》当前定位为 Web 端优先的 2D / 类 2.5D 像素风动作 RPG。项目北极星是：让“工艺”成为动作，让“师承”成为成长，让“归位”成为战斗终点。

首个纵切聚焦“江南雨巷 + 纸伞门径”。当前轻量 demo 用 15-25 分钟的小型区域验证基础移动、闪避、架伞、完美格挡、弹反、风息资源、收风、借风越行、工艺选择和四维反馈；M1 文档与内容数据侧已把江南扩展为第一版投放设计书和数据草案，覆盖主线、人物、子地区、资产、音频和数据结构。

长期技术方向是 C++20/23 + Axmol + WebAssembly/WebGL2 + 自研 Heavenwrights Runtime + Python 内容工具链。工程应保持规则层、内容层和表现层分离：核心战斗、技能、NPC、工艺和四维状态不能写死在具体场景回调里；内容数据需要文本化、可校验、可 diff，并支持 AI 协作。

世界观基调是架空九州承平社会，不是末世或单纯魔法世界。玩家是持临时渡牒和工灯的年轻执灯渡人，通过护送、巡检、试艺、经营、战斗和祟局归位补录《天工录》，让器物、场所和人各得其位。

## 维护规则

- 入口负责导航：启动命令、文档地图和当前状态写在 `README.md` 与本文件。
- 手册负责规则：长期工程边界、目录约定和扩展方式写入 `01_开发者手册.md`。
- 规划负责未来：下一步目标、优先级、验收和不做事项写入 `02_项目规划与验收.md`。
- 全局设计负责准绳：基础设定、地区包、系统边界和数值参考写入 `03_项目全局设计规划.md`。
- 测试负责验收：可复用的回归命令和人工检查写入 `12_测试与验收指南.md`。
- 历史负责沉淀：已完成事实、验证证据和剩余风险写入 `CHANGELOG.md`。
- 架构决策负责取舍：重大技术选择、第三方接入和长期发布目标写入 `doc/adr/`。
