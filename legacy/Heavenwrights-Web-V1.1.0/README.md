# 天工渡 / Heavenwrights（旧版 Web 原型归档）

> 文档定位：旧版项目入口，说明归档版本的可玩范围、运行方式和文档入口。
> 归档版本：V1.1.0 · 江南雨巷阶段可玩节点
> 最后更新：2026-07-10
> 维护状态：功能冻结；新版工程与设计不在本目录继续演化。
> 关联文档：`doc/README.md`、`doc/01_开发者手册.md`、`doc/02_项目规划与验收.md`、`doc/03_项目全局设计规划.md`、`doc/12_测试与验收指南.md`、`CHANGELOG.md`

这是《天工渡》第一代 Web/Canvas 白盒的完整归档。它保留“江南雨巷 + 纸伞门径”的世界观、数据内容库、编辑器草稿与浏览器 Demo，用于回看早期验证结论，不再代表新版端游工程的技术路线。

## 阶段成果

本归档已形成一条可以从开始走到结算的短闭环：

```text
水路入镇
-> 雨棚验货与架伞教学
-> 竹林采集与伞面工艺
-> 夜市委托与风铃铆
-> 旧桥伞影归位
-> 断桥雨幕 Boss
-> 返回纸伞铺完成《天工录》落款
```

玩家现在能看到五段阶段路线；生机归零会进入失败态，可从阶段据点整备并保留已完成的主线、材料与选择；深层菜单会暂停战斗；Boss 归位并落款后会出现明确的阶段完成界面。章节资料库中的“湖田外圩”等扩展草案继续保留，但不会阻塞本次可玩切片结算。

## 快速运行

需要 Node.js 与 Python：

```bash
npm test
npm run validate:content
npm run dev
```

打开：

```text
http://127.0.0.1:4174/
```

当前自动验收基线：

- 52 个 Node 测试。
- 22 个内容 JSON 文件校验。
- 浏览器人工验收覆盖首屏、阶段目标、失败/重试、深层菜单暂停、Boss 归位与完成落款。

## 操作

- `WASD` / 方向键：移动。
- `Shift` / `Space`：冲刺。
- `1`：纸刃；`2`：伞旋。
- `Q`：架伞；`E`：收风；`R`：借风越行。
- `F`：交谈、采集、归位、落款。
- `M` / `I` / `N`：地图、背包、人物。
- `L` / `O`：江南大地图、内容编辑器。
- `T`：小憩推进时辰。

若只需快速验证内容数据闭环，可访问 `http://127.0.0.1:4174/?debug=1`，再在“江南M1”面板使用“推进 M1 流程”；它是白盒捷径，普通入口默认隐藏，也不替代正常游玩验收。

## 归档边界

- 可复用：世界观北极星、执灯渡人、《天工录》、市井/工事/祟局三层空间、稳定 ID、数据/规则/表现分离等设计资产。
- 不再沿用：Web 首发、Axmol + WebAssembly 作为唯一主路线、在单个 Canvas 原型中继续堆叠正式内容。
- 新版不得直接复制 `main-v2.js` 作为 Runtime；若迁移内容，应通过正式 Schema 与转换器导入。
