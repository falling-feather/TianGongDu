# 样例画面忠实度记录

验收基准：`assets/concepts/gameplay-hud-screen.png`。运行截图：`evidence/01-start-screen.png` 至 `04-victory.png`。

| 比较项 | 方向稿 | 实际程序 | 结论 |
| --- | --- | --- | --- |
| 镜头 | 斜向全景，能看见远山、屋面和地面战斗带 | 使用同方向的干净预渲染背景，战斗面限制在画面下半部 | 忠实 |
| 尺度 | 玩家可辨、精英明显大于玩家 | 玩家与精英站高约为同画幅的 1:1.45，并随脚点纵深缩放 | 忠实 |
| 层次 | 前景檐角/雨帘，中景交战，远景渡口机械 | 背景保留三层构图，角色按脚点排序；未实现动态遮挡淡化 | 部分忠实 |
| HUD | 左上玩家、顶端敌人、右上目标、底部四动作 | 位置、墨黑/旧铜/米纸色和主要信息结构一致 | 忠实 |
| 战斗读招 | 精英朱砂预警、重击方向明确、特效克制 | 红圈蓄力、攻击线、闪避“看破”、击退和镜头震动均由程序触发 | 忠实 |
| 战场密度 | 一名精英和三名小敌 | 仅一名精英 | 有意减少，保证最小玩法闭环 |
| 角色动作 | 概念图暗示完整动作姿态 | 单张透明切图通过位移、旋转、缩放、染色表现 | 有意减少，不代表正式动画 |
| 功能范围 | HUD 暗示任务、消耗品和更长战斗 | 只含移动、轻重击、架势、闪避、胜负和重开 | 有意减少，不代表一小时纵切 |

首屏文案差异：方向稿使用“沿伞巷追踪工痕”等完整任务语义；样例改为“击破伞骨镇守”，避免暗示尚未实现的任务链。按钮文案保留轻击、重击、闪避和架势，但去除消耗品与锁定技能。

验证记录：在内置浏览器以 1672×1050 原生窗口检查布局与 WebGL 画面；因 WebGL 连续截图超时，最终四张证据改用本机 Chrome 150 的 Playwright 自动化采集。`view_image` 对照方向稿、战斗态与闪避态完成最终视觉复核；760 px 窄窗口无横向溢出。控制台错误与页面错误均为 0。
