# 15｜服务端、热更新与模组边界

> 上级文档：[`../01-开发者文档.md`](../01-开发者文档.md)
> 状态：Accepted Boundary；1.0 离线优先
> 最后更新：2026-07-10

## 1. 离线优先承诺

1.0 的主线、存档、战斗、工艺、关系和结局在没有账号、没有网络、服务停机时完整可玩。联网能力提升更新、诊断和社区内容体验，但不是单机规则依赖。

客户端不得为了领取本地任务奖励、验证普通物品或推进剧情请求服务器。不存在强联网经济和服务器权威战斗。

## 2. 可选控制面

服务端只考虑：

- 版本/渠道清单与内容包下载地址。
- 发布签名公钥轮换与撤销信息。
- 可选崩溃/性能诊断上传。
- 可选匿名产品遥测。
- 社区模组索引、审核元数据和举报（未来）。
- 官网/公告等非游戏关键服务。

账号、云存档、成就和跨设备同步不在 1.0 必须范围；若加入，必须有离线降级、冲突解决和隐私审查。

## 3. 更新清单

更新清单声明：渠道、发布 ID、可执行版本、内容 API、包集合、文件哈希/大小、完整/差分地址、最低来源版本、强制性、签名、发布时间、回滚目标和发布说明 URL。

客户端流程：

1. 启动后在不阻断离线游玩的超时内获取清单。
2. 验证 TLS 响应之外的清单签名；网络来源不天然可信。
3. 计算兼容路径、磁盘空间和存档风险。
4. 后台下载到隔离目录，支持断点续传和带宽限制。
5. 校验大小、哈希、签名和包依赖。
6. 游戏退出或安全重启时原子切换版本指针。
7. 首次启动失败自动回滚并进入安全模式。

更新不能覆盖正在使用的文件，也不能在未确认时删除唯一旧版本。

## 4. PCK 与内容包

Godot 官方支持运行时加载 PCK/ZIP 资源包，本项目将其视为运输/挂载层，不把安全全部交给格式。包外层仍有项目清单、哈希、签名、依赖、内容 API 和允许路径。

官方内容包只包含数据和资源。任何新增 C# 逻辑要求发布新可执行版本，避免热更新程序集造成安全、AOT、存档和回滚失控。

运行中仅允许加载与当前 Session 无关的预览/语言资源；地区/规则包在标题页或受控切档时挂载。不得在 Boss 战中热替换权威定义。

## 5. 差分与回滚

先实现可靠完整包，再评估差分。差分以旧哈希为前置，应用后验证完整新哈希；任何不匹配回退到完整下载。

安装保留当前与上一可启动版本，以及共享包引用计数。回滚检查存档兼容范围；若新档不可由旧版本读取，提供只读备份/继续使用新版本，不静默丢字段。

渠道：internal → nightly → preview → stable。发布按百分比或人工批次推进，但单机玩家可以在离线状态继续当前版本；安全强制更新需极少使用并说明原因。

## 6. 模组 1.0 边界

允许：

- 新增独立命名空间的数据对象、对话、任务、物品和有限场景资源。
- 使用公开能力、效果、任务节点和 UI 扩展槽。
- 本地化、外观和经过预算限制的音频/视觉资源。
- 在专用模组存档/配置中保存状态。

禁止：

- 加载任意 C#/.NET 程序集、原生 DLL、脚本解释器或系统命令。
- 覆盖核心 ID、官方包文件、签名、公钥、可执行文件和存档头。
- 网络访问、读取任意磁盘、收集个人信息。
- 声称与官方包相同的包 ID/作者身份。
- 绕过内容预算或加载路径限制制造资源耗尽。

模组 ID 使用 `mod.<author>.<package>.<type>.<name>`；资源挂载到隔离前缀。模组默认禁用，玩家显式启用并看到权限/兼容说明。

## 7. 模组存档与兼容

存档头记录启用模组、版本和哈希。加载缺失模组的档时：先分析其拥有的持久状态；可隔离则提示并备份，存在关键依赖则拒绝进入而不删除。

官方迁移不保证理解任意模组内部，但提供公开迁移接口和版本事件。模组状态放命名空间块，禁用后保留墓碑或隔离，不混入官方字段。

“无模组安全模式”始终可从启动器/参数进入，用于诊断，不自动保存覆盖原档。

## 8. 遥测与崩溃诊断

默认原则：最少、透明、可关闭、目的绑定、有限保留。1.0 在未完成隐私审查前不发送产品遥测；崩溃报告也先让玩家查看类别和选择发送。

允许候选数据：构建/包版本、硬件档、匿名 Session 随机 ID、崩溃栈、性能聚合、任务/Boss 阻断事件。禁止默认收集：姓名、邮箱、完整路径、聊天/输入文本、完整存档、模组内容、长期设备指纹。

日志在本地先脱敏；上传有大小限制、重试上限和删除策略。地区盲测时使用单独研究同意，不混同发布遥测。

## 9. 网络实现要求

- 所有响应有大小、时间、重试和解析深度限制。
- URL 不由内容包任意指定；下载域名/证书策略受控。
- 清单和包必须做独立签名验证，密钥支持轮换与撤销。
- 客户端不内置私有签名密钥或服务端秘密。
- 下载文件先写隔离目录，不以响应文件名直接落盘。
- 重定向、代理、离线、限速、磁盘满和中断有测试。
- 服务不可用不阻断离线主菜单和本地存档。

## 10. 服务 API 原则

API 版本化、幂等、缓存友好，错误使用稳定代码。客户端对未知字段宽容，对缺少必需字段严格。服务端不根据客户端自报数据授予单机核心权益。

若未来加入云存档：上传不可覆盖未见过的远端版本；以 revision/etag 检测冲突，保留双份并让玩家选择或提供可解释合并。

## 11. 发布与事故

每次更新保留：清单、签名、包、SBOM、迁移、发布说明、构建日志、回滚步骤和已知问题。发现损坏包时先撤回清单/停止扩散，再发布修复；不要求玩家手工删除存档。

密钥泄露、恶意模组、供应链包和隐私事件分别有处置预案。公开说明区分已确认事实、影响范围、临时措施和后续修复。

## 12. 未来多人边界

若长期版本研究协作玩法，必须新开 ADR 和网络原型。当前确定性命令/事件与实体模型提供基础，但不宣称现有单机 Runtime 自动等于联网架构。不得为不确定多人需求牺牲 1.0 单机体验。

架构原因见 [`adr/ADR-0004-离线优先更新与受限模组.md`](adr/ADR-0004-离线优先更新与受限模组.md)。
