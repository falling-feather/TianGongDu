## 目标

把已确认的“《斗战神》第一指导、2.5D 斜向全景”从概念印象转成可生产、可替换、可验收的 F1 视觉基线：同镜头方案、比例尺、形状/材质语言、技术路线比较、代表资产和完整来源/许可/预算记录。一次性 Phaser 样例只作为方向输入，不是正式资产源。

## 第一阶段交付

1. 用同一“雨夜试伞”斜向全景镜头做 2–3 套原创方案，均包含前景遮挡、中景纵深战斗、远景工艺地标、玩家、两类敌人与证据点；逐项记录保留/否决原因。
2. 产出 720p、1080p 和当前 1230×692 验收画幅的比例尺：玩家、沈砚、伞偶、纸鹭、四时伞祟、门/伞架/工位的站高、脚点、碰撞原点、遮挡层和 Boss 屏占比。
3. 冻结原创形状、色彩和材质关键词：湿青石、旧铜、深木、油纸、绷索、雾雨、暖灯；同时写明不能沿用的参考作品可识别表达。
4. 比较 Sprite 帧/骨骼、预渲染层、低复杂度实时 3D 与混合方案，给出动画产能、下载/显存、透明过绘、锚点/命中和多端缩放的量化建议，不凭偏好锁死路线。
5. 制作一组代表样本：玩家或沈砚、伞偶/纸鹭之一、雨渡/伞巷构件、一次读招 VFX；每项有 source/import/license/runtime/preview/budget metadata。
6. 回写 14 §17 的唯一台账。只有源、导入、运行时预览、许可和预算齐全才允许升级状态。

## 资产与版本纪律

- 正式源文件放入 `assets_src/f1-visual-baseline/` 的稳定子目录；运行时派生产物遵守 14 的生成/缓存边界。
- 大二进制进入 Git 前先核对 Git LFS 配额、跟踪规则和可复现导出；未锁定时只提交小型评审预览与 metadata，不把 DCC 源硬塞普通 Git。
- 使用图像生成时调用项目可用的 imagegen 流程，保存提示约束、派生关系和人工修改记录；AI 生成不豁免许可/来源台账。
- 不下载、描摹或重制《斗战神》《暖雪》资产；第一指导是镜头、尺度、材质重量和场面调度，不是角色/UI/场景复制。

## 验收

- 同一镜头下能清晰判断脚点、纵深、敌我、交互和预警，前景遮挡有淡化/裁切方案。
- 玩家不是俯视小人，精英/Boss 有压迫尺度但不遮蔽危险；关键危险不只靠颜色。
- 代表样本替换现有占位时不要求修改 Gameplay 碰撞、Stable ID 或命中规则。
- 输出对比图、原尺寸预览、资产清单、预算表和评审结论；证据绑定角色分支 commit。

## 非目标

本阶段不做三地区最终资产、全套角色动画、完整音频、发布包或正式资产方案大规模扩产；未通过视觉/技术基线评审前不得批量生产。
