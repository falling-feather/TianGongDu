## 0. 文档边界

本文件在原“江南雨巷 + 纸伞门径”纵切规格基础上扩展为第一版投放设计书。前 1-9 章保留 M0/M1 共用的纵切骨架，10 章之后补充 M1 第一版投放所需的全量剧情、人物、子地区、数据结构和资源生产计划。

## 1. 纵切定位

名称：江南雨巷 + 纸伞门径。

体验时长：15-25 分钟。

形态：

- Web 端类 2.5D 像素动作区域。
- 城镇据点承接 NPC、工艺、商店、任务。
- 地区关卡承接战斗、采集、机关、支路、Boss。
- 类 2.5D 用于斜俯视动作平面、Y 轴排序、前景遮挡、雨幕、屋檐、桥影和水巷景深。

## 2. 地区结构

| 节点 | 类型 | 功能 |
| --- | --- | --- |
| 水路入镇 | 护送小段 | 护送伞骨与桐油，展示执灯基础战斗 |
| 镇口雨棚 | 教学小段 | 平面移动、闪避、第一只雨魇 |
| 水巷主路 | 街巷主路 | 轻击、闪避、居民反馈 |
| 纸伞铺 | 据点房间 | 师傅 NPC、工艺链、技能解锁 |
| 竹林小径 | 采集支路 | 选竹、地形穿行、小怪伏击 |
| 夜市口 | 据点 / 选择 | 商业急单、四维选择 |
| 旧桥下层 | 战斗房间 | 精英怪、伞师循环检验 |
| 屋檐上层 | 回访支路 | 学会借风越行后到达 |
| 断桥雨幕 | Boss 房间 | 雨巷破伞执念 |

空间原则：

- 主路清晰，支路少而有意义。
- 首次经过时能看到上层奖励，但暂时到不了。
- 学会借风越行后，通过风口、水面和屋檐落点到达上层或侧路。
- 关卡不做迷路型大地图，先做 15-25 分钟密度。

## 3. 类 2.5D 层级

```text
FarBackground: 远山、水雾、屋瓦
BackDecor: 店招、灯笼、桥影、水面反光
GroundPlane: 可行走地面、台阶、桥面、浅水边界
Gameplay: 玩家、敌人、机关、可交互物
FrontDecor: 雨棚、檐角、近景竹叶、桥栏遮挡
Weather: 雨线、水花、风线、湿气
UI: HUD、交互提示
```

战斗碰撞只发生在清晰的 2D 逻辑平面。场景表现允许前后遮挡、Y 轴排序和高度暗示，但不能让视觉深度影响判定清晰度。

## 5. 伞师类 2.5D 动作

| 能力 | 用途 |
| --- | --- |
| 架伞 | 正面防御，抵挡雨针和伞刺 |
| 完美格挡 | 减少气力消耗，获得风息 |
| 弹反 | 打出破绽，可接轻击、重击或收风 |
| 收风 | 拉近轻敌、吹散雨幕、触发风铃机关 |
| 伞旋 | 小范围控场，适合被围时使用 |
| 借风越行 | 短距离越过水面、风口或屋檐落点，用于探索回访和战斗换位 |

纵切技能节点：

- 稳骨：格挡气力消耗降低。
- 吃风：完美格挡获得风息。
- 收风：主动拉近 / 打断 / 积破绽。
- 借风：短距离越行 / 滞空，服务探索和换位。

## 6. 工艺链

| 工序 | 类 2.5D 玩法 | 战斗 / 探索映射 |
| --- | --- | --- |
| 选竹 | 竹林支路观察竹节，选择材料 | 伞骨稳定，影响格挡气力 |
| 削骨 | 简短力度控制，不做复杂 QTE | 影响破损概率 |
| 糊纸 | 控制张力和覆盖 | 影响风息获取和湿滞抗性 |
| 试伞 | 在风口、水面和屋檐落点开合伞面 | 解锁借风稳定度 |

## 8. 四维选择

夜市商业急单：

| 选择 | 结果 |
| --- | --- |
| 接急单并省工 | 市场 +15，传承 -10，精神 -5 |
| 拒绝省工，坚持完整工序 | 传承 +15，精神 +10，市场 -5 |
| 寻找替代材料折中 | 市场 +5，传承 +5，生活 +5，但增加探索步骤 |

至少影响：

- 师傅最后评价。
- 夜市居民台词。
- Boss 场景修复后的视觉状态。

## 9. 验收标准

- 从新游戏到 Boss 结束可连续游玩。
- 不依赖开发者口头解释也能理解主线。
- 伞师门径与普通近战有明显差异。
- 工艺链至少影响一个战斗参数和一个外观 / 反馈。
- 四维选择至少影响三个后续反馈点。
- Boss 战能验证格挡、弹反、收风、借风越行。

## 10. M1 第一版投放目标

M1 第一版投放要把江南雨巷从首屏 demo 推进为可拆任务、可做白盒、可开资产、可建数据的地区包。它不要求一次性实现全部内容，但必须让程序、策划、美术、音乐和 QA 都能知道下一步该生产什么。

### 10.1 体验目标

| 目标 | 说明 |
| --- | --- |
| 地区完整感 | 雨巷不是一条线性关卡，而是由水路、主巷、作坊、竹林、晒场、夜市、旧桥和祟局组成的生活空间。 |
| 人物差异 | 每个关键 NPC 有职业、性格、关系、支线和四维倾向，不能只做功能按钮。 |
| 工艺可玩 | 纸伞工序至少影响一个战斗参数、一个探索能力、一个视觉反馈和一个 NPC 评价。 |
| 选择有回声 | 夜市选择、材料选择和 Boss 归位方式要改变台词、场景、账册或后续回访。 |
| 数据可生产 | 设计能拆成稳定 ID 的内容对象，后续可进入 JSON/CSV/Markdown 管线。 |

### 10.2 推荐时长

| 版本 | 目标时长 | 范围 |
| --- | ---: | --- |
| 最小纵切 | 15-25 分钟 | 沿用前文 M0 目标，从入镇到 Boss 归位。 |
| M1 核心投放 | 25-35 分钟 | 完成 7 个核心子地区和主线。 |
| M1 完整投放 | 45-60 分钟 | 完成主线、3 条以上支线和至少 1 次回访。 |

### 10.3 核心情绪

江南雨巷不是阴郁灾区，而是雨季将至、灯市将开、商路繁忙、工坊兴盛的水镇。玩家应先感到地方有生活，再看到生活被订单、工序、桥况和祟局牵动。

关键词：温润雨声、水路货声、纸伞开合、窄巷灯影、竹林风口、夜市人潮、旧桥回响、归位后的安静。

## 11. 主线剧情扩展

### 11.1 一句话梗概

雨季前夕，江南雨巷同时收到远方伞单和本地灯市需求。年轻作坊希望压缩工序提高产量，老伞师坚持完整试伞规矩。玩家作为执灯渡人护送伞骨与桐油入镇，在纸伞铺学艺、竹林选材、夜市见账、旧桥巡检中理解雨巷的真实矛盾，最终进入断桥雨幕，让破伞祟局归位，并把自己的判断写入《天工录》。

### 11.2 起承转合

| 阶段 | 时间 | 目标 | 重点体验 |
| --- | ---: | --- | --- |
| 起：水路入镇 | 0-6 分钟 | 护送伞骨与桐油，抵达雨巷 | 基础移动、轻击、闪避、工灯照形、第一只雨魇。 |
| 承：拜师试伞 | 6-18 分钟 | 见沈雨，理解订单压力，学习架伞 | 师承、格挡、工序、材料选择。 |
| 转：夜市分歧 | 18-30 分钟 | 远方急单与完整试伞发生冲突 | 四维选择、NPC 关系、旧桥异常前兆。 |
| 合：断桥归位 | 30-45 分钟 | 进入祟局层，平复破伞执念 | 完整战斗循环、收风、借风、Boss 归位。 |
| 回声：入录落款 | 45-60 分钟 | 结算后果，开启回访与下一地区线索 | 师傅评价、夜市变化、工灯校准、天工录记录。 |

### 11.3 主线章节

| 章 | 标题 | 主要地点 | 剧情目标 | 玩法目标 |
| ---: | --- | --- | --- | --- |
| 1 | 雨线未至 | 水路入镇、镇口雨棚 | 护送材料入镇，确认伞单来源。 | 基础战斗和护送。 |
| 2 | 伞骨有声 | 纸伞铺、竹林小径 | 见沈雨，查伞骨质量，认识学徒分歧。 | 架伞、选竹、削骨。 |
| 3 | 纸面吃风 | 染纸晒场、风铃巷 | 糊纸、上油、试伞，找到雨幕异响。 | 收风、雨幕机关、风向谜题。 |
| 4 | 灯市有账 | 夜市口、行账棚 | 处理远方急单与本地灯市冲突。 | 四维选择、NPC 说服。 |
| 5 | 旧桥压雨 | 旧桥下层、屋檐上层 | 巡桥、取风铃铆、确认祟局入口。 | 借风越行、回访支路。 |
| 6 | 断伞归位 | 断桥雨幕 | 与雨巷破伞执念交锋并归位。 | Boss 三阶段、归位交互。 |
| 7 | 落款入录 | 纸伞铺、夜市、祠前 | 评价选择后果，记录《天工录》。 | 地区回声、回访解锁。 |

### 11.4 Boss 归位逻辑

雨巷破伞执念不是“纸伞怪物”，而是旧桥雨棚下积存的废伞、失准工序、赶工焦虑、灯市期待和旧桥潮湿风声交织成的祟局。

归位流程：

1. 打散外层雨针与破伞影，露出裂伞核。
2. 用架伞、弹反和收风压低失序。
3. 借风越行到风口位置，校正雨幕方向。
4. 根据玩家选择使用不同归位动作。
5. 归位后进入旧桥现实层，工灯记录伞骨纹、试伞风口和无款工印线索。

| 路径 | 条件 | 后果 |
| --- | --- | --- |
| 速成交付 | 接急单并省工 | 夜市货期保住，旧桥仍保留细裂，师傅评价带疑虑。 |
| 完整工序 | 坚持完整试伞 | 旧桥雨幕最稳，灯市摊位减少，行账房对货期不满。 |
| 折中补修 | 完成竹林/染纸补材支线 | 四维均衡，流程较长，阿青获得成长台词。 |

## 13. 人物体系扩展

### 13.1 角色设计原则

江南 NPC 至少承担两个身份：

- 功能身份：教学、商店、材料、支线、Boss 见证、四维反馈。
- 情感身份：师傅、学徒、跑单人、居民、守桥人、灯会组织者、旧伞修补人。

NPC 不能只围绕玩家转。每个人都应有自己的工作、行程、立场和对雨巷的具体利益。

## 16. 敌人、机关与 Boss 扩展

### 16.1 敌人设计原则

敌人是失序、器影、雨幕和工事问题的动作化表达，不是把真实工艺或地方文化妖魔化。

每种敌人必须服务一个学习点：

- 雨魇：移动、轻击、闪避。
- 破伞影：格挡和弹反。
- 竹壳影：地形和收风。
- 风回声：音色机关和方向提示。
- 断骨伞灵：完整伞师循环。
- Boss：地区全部能力汇总。

## 21. 内容生产顺序

M1 推荐顺序：

1. 锁定主线任务树和 7 个核心子地区。
2. 建立 `SubregionDef / QuestDef / BuildingDef / InteractionDef` Schema 草案。
3. 扩展 NPC 到 8 名核心角色。
4. 扩展敌人到 5 类和 1 个 Boss。
5. 建作坊、竹林、夜市、旧桥、断桥雨幕的白盒流程。
6. 补美术概念和音频占位。
7. 把四维选择接入至少 3 个反馈点。
8. 进行 45 分钟人工流程审查。

M1 最小核心：

- 子地区：水路入镇、水巷主路、作坊、竹林、夜市、旧桥、断桥雨幕。
- NPC：沈雨、骆青禾、阿棠、周桐、阿青、叶郝、梅守桥、老杜。
- 敌人：雨魇、破伞影、竹壳影、断骨伞灵、Boss。
- 选择：急单、省工、完整工序、折中补修。

## 22. 验收标准扩展

### 22.1 设计验收

- 主线从入镇到 Boss 归位可用步骤表清楚描述。
- 至少 7 个核心子地区有玩法、风格、建筑、NPC、敌人/机关和回访价值。
- 完整规划至少 10 个子地区，不同子地区不能只靠色调区分。
- 至少 12 名 NPC 有身份、性格、功能、关系、支线和四维倾向。
- 至少 3 条支线能影响主线反馈或 Boss 归位结果。
- 工艺链至少影响一个战斗参数、一个外观反馈和一个 NPC 评价。
- Boss 归位不是击杀，必须有归器礼或等价交互。

### 22.2 数据验收

- 所有新对象有稳定英文 ID。
- 中文文本不写入逻辑对象，使用 `displayNameKey`。
- Region 能引用 Subregion、Quest、Npc、Enemy、WorldEvent、Pathway。
- Quest 能引用步骤、选择和奖励。
- Interaction 能引用触发者、目标和效果。
- BossPhase 能引用 Boss、技能、VFX、阶段切换条件。

### 22.3 生产验收

- 程序能据此建立 Schema 和加载计划。
- 策划能据此拆任务树、数值表和交互表。
- 美术能据此开概念图、角色、敌人、建筑和动画需求。
- 音乐能据此开 BGM、环境音、技能音效和机关提示。
- QA 能据此建立文档验收和后续 playable 验收清单。

### 22.4 自动化验收

当前只要求文档阶段通过：

```bash
npm test
npm run validate:content
```

后续内容数据扩容后，需要增加：

- 跨文件 ID 引用检查。
- 本地化 key 缺失检查。
- 数值上下限检查。
- 任务步骤可达性检查。
- 子地区连接图检查。
- BossPhase 转换条件检查。
- 资产 manifest 引用检查。

## 23. 资料核验与风险

### 23.1 已参考方向

- 非遗表达应以活态实践、知识、技能、器物和文化空间为核心，而不是把工艺做成静态图鉴或怪物。
- 国家级非遗名录以项目、类别、地区、保护单位等方式组织资料，后续查证纸伞、伞制作技艺、江南水巷和地方工艺时应优先使用官方名录、地方文旅/非遗中心和可靠出版物。
- 纸伞工序当前只作为玩法占位，正式文本前需要核验具体地域工序差异。

优先资料入口：

- 中国非物质文化遗产网：`https://www.ihchina.cn/`
- UNESCO Intangible Cultural Heritage：`https://ich.unesco.org/`

### 23.2 待核验清单

| 主题 | 需要核验 |
| --- | --- |
| 纸伞工序 | 选竹、削骨、穿线、糊纸、上油、晾晒、绘制、试伞等是否适用于目标地域表达。 |
| 材料 | 竹、纸、桐油、染料、线材的真实称谓和地域差异。 |
| 江南水镇空间 | 码头、桥、埠头、夜市、灯会、雨棚、作坊布局。 |
| 行会与账房 | 工坊、行会、货栈、跑单、账册等称谓是否自然。 |
| 仪式与灯市 | 祠灯、风铃、旧桥归位等表达是否过度虚构。 |
| 敌人命名 | 避免把真实文化项目、真实传承人或地方信仰直接敌对化。 |
| 美术符号 | 避免滥用真实宗教符号或地域刻板符号。 |

### 23.3 表达风险控制

- 不使用真实传承人姓名做 NPC。
- 不把真实非遗项目名称直接作为敌人名称。
- 不把“赶工”和“创新”写成单纯负面，叶郝等革新派必须有合理性。
- 不把“守旧”和“完整工序”写成绝对正确，沈雨也要承认生计压力。
- 不让 Boss 代表某种文化本身，Boss 只代表具体事件中器物、风雨、订单和记忆的失序。
